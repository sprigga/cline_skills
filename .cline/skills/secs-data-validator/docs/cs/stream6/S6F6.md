# S6F6 - Multi-block Grant

**Direction:** Sent by Host Only

---

## Format

**Message Structure:** Host grants or denies permission for multi-block data transfer (reply to S6F5R).

### Format Explanation

The **Format** shows a simple single-item message:

```
[GRANT6]
```

- **`[GRANT6]`** — Grant Code (B:1) — a single binary byte
  - `0` = GRANT — allow multi-block transfer
  - `1` = BUSY — host is busy, retry later
  - `2` = DENY — host denies the transfer

### SECS Notation Reference
- `[ITEMID]` = A single data item
- `B:1` = Binary item, 1 byte

---

## Codebase

**Message Type:** Reply (sent in response to S6F5R)

### Codebase Explanation

This C# code demonstrates how to **send and parse a reply** to S6F5R (S6F6) using the **secs4net** library.

#### Key Components

1. **Sending the Message** (line 14)
   - `sp.SendSecsMsg(6, 5, ...)` sends S6F5R to the equipment and waits for the reply (S6F6)
   - Returns `RcResult` containing the response in `rcr.result`

2. **Handler Signature** (lines 16-19)
   - This is **reply parsing**, not a receive handler
   - `rcr.rc` = Return code (0 = success)
   - `TSN_data` = The reply message data as a string

3. **Parsing Logic**
   - **Line 24**: `ListSplit(TSN_data)` breaks the message into elements:
     - `[0]` = "B:1" (binary type, 1 byte)
     - `[1]` = The actual value (binary string)

   - **Line 27**: `sp.BinToInt(list0[1])` converts the binary value to integer
     - 0 = GRANT (transfer allowed)
     - 1 = BUSY (retry later)
     - 2 = DENY (not allowed)

#### Parsing Workflow
1. Send the message with `SendSecsMsg()` and wait for reply
2. Check return code `rcr.rc` for communication errors
3. Split reply data → check first element is "B:1"
4. Convert binary to integer for the grant code

#### Common Grant Codes
| Code | Meaning | Action |
|------|---------|--------|
| 0 | GRANT — proceed with transfer | Start receiving multi-block data |
| 1 | BUSY — host is busy | Retry the request later |
| 2 | DENY — not allowed | Abort the data transfer |

```
    // S6F6 C# Parse reply
    SecsPort sp=(SecsPort)your_SecsPort
    RcResult rcr = sp.SendSecsMsg(6, 5, true, sendData, true);
    string TSN_data;
    bool ok=true;
    while (ok) {   // break out of loop on error
        if (rcr.rc != 0) {ok=false; break;}
        TSN_data = rcr.result;
        // expect GRANT6
        // variables for data items and parsing
        int GRANT6;  	// B:1 (always)  multblock permission grant
        string [] list0;
        list0 = SecsPort.ListSplit(TSN_data);
        if (list0.Length < 1) { ok = false; break; }
        if (list0[0] != "B:1") { ok = false; break; }
        GRANT6 = sp.BinToInt(list0[1]);
        return;  // parsed ok
        } // end while(ok)
```
