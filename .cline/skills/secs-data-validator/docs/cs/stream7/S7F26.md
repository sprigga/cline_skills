# S7F26 - Formatted Process Program Data

**Direction:** Sent by Host and Equipment

---

| **S7F26** | Formatted Process Program Data | Sent by Host and Equipment |

Comment: Reply to S7F25R. Contains the formatted process program data with its ID, equipment model info, and command/parameter structure.

### Format (TSN)

```
{L:4
  [PPID]
  [MDLN]
  [SOFTREV]
  {L:c
    {L:2
      [CCODE]
      {L:p
        [PPARM]
      }
    }
  }
}
```

**Format Explanation:**

這是一個**四層巢狀結構**，與 S7F23R 的格式完全相同，用來回覆格式化製程程式的完整資料：

- **`{L:4}`** — 外層 list，固定包含 4 個元素：
  1. **`[PPID]`** — Process Program ID（製程程式 ID），型別 `A:80`（ASCII 字串，最大 80 字元）。回覆的製程程式名稱。
  2. **`[MDLN]`** — Equipment Model Type（設備型號），型別 `A:20`（固定 20 字元）。此製程程式適用的設備型號。
  3. **`[SOFTREV]`** — Software Revision（軟體版本），型別 `A:20`（固定 20 字元）。此製程程式適用的軟體版本。
  4. **`{L:c}`** — 命令清單 list，包含 `c` 個命令（`c` 為變數），每個命令是一個 `{L:2}` 結構：
     - **`[CCODE]`** — Process Operation Command Code（製程操作命令碼），型別 `A:n`（可變長度 ASCII 字串）。
     - **`{L:p}`** — 該命令的參數 list，包含 `p` 個參數（`p` 為變數）：
       - **`[PPARM]`** — Process Parameter（製程參數），型別 `A:n`（可變長度）。可以是任意純量或向量型別。

### CCODE 的用意與設計目的

S7F26 回覆中的 CCODE（Process Operation Command Code，製程操作命令碼）是格式化製程程式的**核心識別單元**。每個 CCODE 代表設備可執行的一個操作步驟（例如 `TEMP_SET`、`GAS_FLOW`），搭配零或多個 PPARM 參數構成一條完整命令。

> **程式比喻：** CCODE 是函式名稱，PPARM 是引數列表。`TEMP_SET(300.0)` → `{L:2  A "TEMP_SET"  {L:1  F4 300.0}}`

#### 為什麼需要 CCODE？

SEMI E5 標準定義了兩種製程程式傳輸方式。CCODE 是格式化（Formatted）與非格式化（Unformatted）的根本差異：

| 特性 | **Formatted**（含 CCODE） | **Unformatted**（純文字） |
|------|--------------------------|--------------------------|
| 訊息 | S7F25R / S7F26 | S7F5R / S7F6 |
| Host 能否解析 | ✅ 語意級別 — 每個命令和參數有明確型別與意義 | ❌ 不透明 — Host 只看到一坨文字 |
| Host 能否驗證 | ✅ 可逐一檢查命令是否合法、參數是否在合理範圍 | ❌ 只能 byte-by-byte 比對 |
| Host 能否修改 | ✅ 可修改單一 PPARM 而不需重傳整份配方 | ❌ 必須由設備端編輯 |

CCODE 解決了非格式化配方的三大痛點：

1. **不透明（Opaque）：** 非格式化配方是黑盒子，Host 無法知道配方裡設定了哪些參數。CCODE 讓每個步驟變成具名、有型別的結構化資料。
2. **不可驗證（Non-verifiable）：** CCODE 搭配 S7F22（Process Capabilities Data）的元資料，讓接收方能驗證命令是否合法、參數型別是否正確、命令順序是否符合約束。
3. **不可攜（Non-portable）：** CCODE 所在的 `L:4` 結構包含 `MDLN` 和 `SOFTREV`，讓接收方能判斷配方是否適用於自己的機型。

#### CCODE 在 S7F26 中的角色

S7F26 是 S7F25R 的回覆，Host 透過它**取回**設備上的配方。CCODE 在此的意義：

- **配方稽核：** Host 收到 S7F26 後，可以逐一檢查每個 CCODE 和 PPARM，確認與 MES 資料庫中的配方一致。
- **Round-trip 驗證：** Host 用 S7F23R 上傳配方後，再用 S7F25R 讀回（得到 S7F26），比對兩者是否完全相同。
- **配方遷移：** 從舊設備讀取配方（S7F26），再用 S7F23R 傳至新設備。CCODE + MDLN + SOFTREV 確保相容性。

#### CCODE 使用規則

1. **CCODE 由設備廠商定義：** SEMI E5 只定義格式（`A:n` + `{L:p PPARM}`），CCODE 的名稱和語意由設備規格書（ECS）定義。
2. **命令清單有序：** `{L:c}` 中的 CCODE 順序即為執行順序。
3. **允許重複：** 同一 CCODE 可出現多次（多步製程中每步都可能有溫度設定）。
4. **PPARM 型別須一致：** 同一 CCODE 的參數型別在整條產線應保持一致。

> **更完整的 CCODE 分析（含 S7F22 元資料與順序約束）請參見 [S7F23R 文件](S7F23R.md#ccode-深度解析)。**

### 結構圖示

```
L:4 ─┬─ PPID (製程程式 ID)
      ├─ MDLN (設備型號)
      ├─ SOFTREV (軟體版本)
      └─ L:c (命令清單) ─┬─ L:2 ─┬─ CCODE (命令碼)
                                   └─ L:p ─┬─ PPARM (參數)
                                            ├─ PPARM
                                            └─ ...
                         └─ L:2 ─┬─ CCODE
                                   └─ L:p ─┬─ PPARM
                                            └─ ...
```

### C# 解析程式

以下 C# 程式碼是**發送端**接收 S7F26 回覆後的解析邏輯，使用 `secs4net` 函式庫：

### 處理流程

1. **發送請求** — 先以 `sp.SendSecsMsg(7, 25, true, sendData, true)` 發送 S7F25R，並等待回覆。
2. **檢查回傳碼** — `rcr.rc != 0` 表示通訊層面失敗，需中斷處理。
3. **解析外層 L:4** — 用 `SecsPort.ListSplit()` 拆解 TSN 資料，驗證外層 list 為 `L:4`。
4. **取出 PPID / MDLN / SOFTREV** — 分別從 `list0[1]`、`list0[2]`、`list0[3]` 解析三個固定欄位。MDLN 和 SOFTREV 需額外驗證型別以 `A:` 開頭。
5. **迴圈解析命令清單** — `list0[4]` 是 `{L:c}` 命令清單，用 `for` 迴圈（變數 `c`）逐一處理每個命令。
6. **解析每個 L:2 命令** — 每個命令包含 CCODE 和參數 list `{L:p}`，再用內層 `for` 迴圈（變數 `p`）取出每個 PPARM。

```csharp
    // S7F26 C# Parse reply
    // sp is a SecsPort or SecsHost object reference
    RcResult rcr = sp.SendSecsMsg(7, 25, true, sendData, true);
    string TSN_data;
    bool ok=true;
    while (ok) {   // break out of loop on error
        if (rcr.rc != 0) {ok=false; break;}
        TSN_data = rcr.result;
        // expect L:4 PPID MDLN SOFTREV {L:c {L:2 CCODE {L:p PPARM}}}
        // variables for data items and parsing
        string PPID;  	// A:80 (varies)  process program ID
        string MDLN;  	// A:20 (always)  equipment model type
        string SOFTREV;  	// A:20 (always)  software revision
        string CCODE;  	// A:n (varies)  process operation command code
        string PPARM;  	// A:n (varies)  process parameter, any scalar or vector
        string [] list0;
        list0 = SecsPort.ListSplit(TSN_data);
        if (list0.Length != 5 || list0[0] != "L:4") { ok=false; break; }
        string [] list1;
        list1 = SecsPort.ListSplit(list0[1]);
        if (list1.Length < 1) { ok = false; break; }
        if (list1.Length == 2) { PPID = list1[1];} else { PPID = ""; }
        list1 = SecsPort.ListSplit(list0[2]);
        if (list1.Length < 1) { ok = false; break; }
        if (!list1[0].StartsWith("A:")) { ok = false; break; }
        if (list1.Length == 2) { MDLN = list1[1];} else { MDLN = ""; }
        list1 = SecsPort.ListSplit(list0[3]);
        if (list1.Length < 1) { ok = false; break; }
        if (!list1[0].StartsWith("A:")) { ok = false; break; }
        if (list1.Length == 2) { SOFTREV = list1[1];} else { SOFTREV = ""; }
        list1 = SecsPort.ListSplit(list0[4]);
        if (list1.Length < 1) { ok=false; break; }
        int c;
        for(c=1; c < list1.Length; c++) {
            string [] list2;
            list2 = SecsPort.ListSplit(list1[c]);
            if (list2.Length != 3 || list2[0] != "L:2") { ok=false; break; }
            string [] list3;
            list3 = SecsPort.ListSplit(list2[1]);
            if (list3.Length < 1) { ok = false; break; }
            if (list3.Length == 2) { CCODE = list3[1];} else { CCODE = ""; }
            list3 = SecsPort.ListSplit(list2[2]);
            if (list3.Length < 1) { ok=false; break; }
            int p;
            for(p=1; p < list3.Length; p++) {
                string [] list4;
                list4 = SecsPort.ListSplit(list3[p]);
                if (list4.Length < 1) { ok = false; break; }
                if (list4.Length == 2) { PPARM = list4[1];} else { PPARM = ""; }
                }
            if (!ok) break;
            }
        if (!ok) break;
        return;  // parsed ok
        } // end while(ok)
```
