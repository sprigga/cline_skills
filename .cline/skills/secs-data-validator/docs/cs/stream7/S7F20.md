# S7F20 - Current Process Program Data

**Direction:** Equipment Only

---

| **S7F20** | Current Process Program Data | Equipment Only |

Comment: Reply to S7F19R. Contains the list of current process program IDs on equipment.

### Format (TSN)

```
{L:n
  [PPID]
}
```

**Format Explanation:**
- **`{L:n}`** — 一個 list 容器，包含 `n` 個 PPID 項目。`n` 為目前設備上所有製程程式的數量。若無任何製程程式，則為 `L:0`。
- **`[PPID]`** — Process Program ID（製程程式 ID），型別為 `A:80`（ASCII 字串，最大 80 字元）。每個 PPID 代表設備上存在的一個製程程式。

### 範例

| 傳送內容 | 意義 |
|---------|------|
| `L:0` | 設備上沒有任何製程程式 |
| `L:3 {A:RECIPE_A} {A:RECIPE_B} {A:RECIPE_C}` | 設備上有 3 個製程程式 |

### C# 解析程式

以下 C# 程式碼是**發送端 (Host)** 接收 S7F20 回覆後的解析邏輯，使用 `secs4net` 函式庫：

### 處理流程

1. **發送請求** — 先以 `sp.SendSecsMsg(7, 19, true, sendData, true)` 發送 S7F19R，並等待回覆。
2. **檢查回傳碼** — `rcr.rc != 0` 表示通訊層面失敗，需中斷處理。
3. **解析外層 list** — 用 `SecsPort.ListSplit()` 拆解 TSN 資料，`list0[0]` 為 `L:n`。
4. **迴圈取出 PPID** — 從 `list0[1]` 開始，每個元素透過 `ListSplit()` 取得實際 PPID 字串值。若 `list0` 只有 1 個元素（`L:0`），表示沒有製程程式。

```csharp
    // S7F20 C# Parse reply
    SecsHost sp=(SecsHost)your_SecsHost;
    RcResult rcr = sp.SendSecsMsg(7, 19, true, sendData, true);
    string TSN_data;
    bool ok=true;
    while (ok) {   // break out of loop on error
        if (rcr.rc != 0) {ok=false; break;}
        TSN_data = rcr.result;
        // expect L:n PPID
        // variables for data items and parsing
        string PPID;  	// A:80 (varies)  process program ID
        string [] list0;
        list0 = SecsPort.ListSplit(TSN_data);
        if (list0.Length < 1) { ok=false; break; }
        int n;
        for(n=1; n < list0.Length; n++) {
            string [] list1;
            list1 = SecsPort.ListSplit(list0[n]);
            if (list1.Length < 1) { ok = false; break; }
            if (list1.Length == 2) { PPID = list1[1];} else { PPID = ""; }
            }
        if (!ok) break;
        return;  // parsed ok
        } // end while(ok)
```
