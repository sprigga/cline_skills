# S1F13R - Establish Communications Request

**Direction:** Different Host and Equipment Use

---

| **S1F13R** | Establish Communications Request | Different Host and Equipment Use |

## Format 說明

S1F13R 是 SECS-II 的「建立通訊請求」訊息，Equipment → Host 和 Host → Equipment 雙向都會使用，但資料格式不同。

在 TSN (Tachyon Standard Notation) 中：
- `L:n` 表示含 n 個元素的清單
- `A:n` 表示最多 n 個字元的 ASCII 字串
- `B:1` 表示 1-byte 二進位值

### Equipment Send Format

```
{L:2
[MDLN](http://hume.com/secs/items.html#MDLN)

[SOFTREV](http://hume.com/secs/items.html#SOFTREV) }
```

設備主動告知主機自己的型號名稱 (MDLN, A:20 字串) 和軟體版本 (SOFTREV, A:20 字串)。

### Host Send Format

```
L:0
```

主機發送時使用空清單，因為主機端不需要提供 MDLN/SOFTREV。

## Codebase 說明

以下 C# 程式碼展示 `secs4net` 函式庫的 S1F13R 接收處理模式：

1. **註冊 handler** — `sp.MessageTypeAdd(1, 13, handler)` 將回呼函式綁定到 S1F13 訊息
2. **TSN 解析** — `SecsPort.ListSplit(TSN_data)` 將 TSN 字串拆分為陣列，逐層檢查 `L:n` 清單結構和資料型別 (`A:` 前綴)
3. **回覆** — 若 `send_reply=True`，用 `sp.SendReply(1, 14, transID, reply)` 回覆 S1F14
4. **錯誤處理** — 若解析失敗且是 Equipment 端，發送 `sp.SendS9(7, header)` (S9F7 illegal data) 錯誤訊息；Host 端不發送 S9F7

### Equipment Receive Handler

Host 收到 Equipment 的 S1F13R 時的處理流程：解析 `L:2 MDLN SOFTREV`，提取設備型號與軟體版本。

### Equipment Send Format (Host Receive Code)

```csharp
// S1F13R C# Receive Equipment message - add next line to setup
//SecsHost sp=your_SecsHost;
sp.MessageTypeAdd( 1, 13, new SecsMsgReceiveDelegate(recv_S1F13R));

// receive method
void recv_S1F13R(object sender, int stream, int function, bool send_reply, int transID, string TSN_data, string header) {
    SecsHost sp = (SecsHost)sender;
    bool ok=true;
    while (ok) {   // break out of loop on error
        // expect L:2 MDLN SOFTREV
        // variables for data items and parsing
        string MDLN;  	// A:20 (always)  equipment model type
        string SOFTREV;  	// A:20 (always)  software revision
        string [] list0;
        list0 = SecsPort.ListSplit(TSN_data);
        if (list0.Length != 3 || list0[0] != "L:2") { ok=false; break; }
        string [] list1;
        list1 = SecsPort.ListSplit(list0[1]);
        if (list1.Length < 1) { ok = false; break; }
        if (!list1[0].StartsWith("A:")) { ok = false; break; }
        if (list1.Length == 2) { MDLN = list1[1];} else { MDLN = ""; }
        list1 = SecsPort.ListSplit(list0[2]);
        if (list1.Length < 1) { ok = false; break; }
        if (!list1[0].StartsWith("A:")) { ok = false; break; }
        if (list1.Length == 2) { SOFTREV = list1[1];} else { SOFTREV = ""; }
        if (send_reply) {
            string reply; // TBD create reply
            sp.SendReply(1, 14, transID, reply);
            }
        return;
        } // end while(ok)
    } // end recv_S1F13R
```

### Host Send Format (Equipment Receive Code)

Equipment 收到 Host 的 S1F13R 時的處理流程：解析空清單 `L:0`，因為 Host 端不需提供設備識別資訊。

```csharp
// S1F13R C# Receive Host message - add next line to setup
//SecsPort sp=your_SecsPort;
sp.MessageTypeAdd( 1, 13, new SecsMsgReceiveDelegate(recv_S1F13R));

// receive method
void recv_S1F13R(object sender, int stream, int function, bool send_reply, int transID, string TSN_data, string header) {
    SecsPort sp = (SecsPort)sender;
    bool ok=true;
    while (ok) {   // break out of loop on error
        // expect L:0
        // variables for data items and parsing
        string [] list0;
        list0 = SecsPort.ListSplit(TSN_data);
        if (list0.Length != 1 || list0[0] != "L:0") { ok = false; break; }
        // single token data such as L:0 or U4:0 has been received as expected
        if (send_reply) {
            string reply; // TBD create reply
            sp.SendReply(1, 14, transID, reply);
            }
        return;
        } // end while(ok)
    // bad data
    sp.SendS9(7, header);
    } // end recv_S1F13R
```

## 實測 Log 案例（KA01 Schmoll 鑽孔機，2026-04-14）

### 場景：通訊中斷後重新建立（雙向 S1F13 握手）

此日誌展示 HSMS 通訊重建時，Equipment 與 Host **雙向**發送 S1F13R 的完整流程，兩次交換在 23ms 內完成。

#### Step 1：Equipment 主動發送 S1F13R（含 MDLN/SOFTREV）

```xml
<SECSMessage s="1" f="13" direction="E to H" replyBit="true"
             txid="5" time="2026-04-14T13:04:13.355Z" deviceID="0">
    <SECSData>
        <LST>
            <ASC>MXY2-200-CCD</ASC>  <!-- MDLN -->
            <ASC>00.01</ASC>         <!-- SOFTREV -->
        </LST>
    </SECSData>
</SECSMessage>
<SECSMessage s="1" f="14" direction="H to E" replyBit="false"
             txid="5" time="2026-04-14T13:04:13.358Z" deviceID="0">
    <SECSData>
        <LST>
            <BIN>0</BIN>   <!-- COMMACK=0: 接受 -->
            <LST />        <!-- MDLN/SOFTREV 空列表（Host 端格式） -->
        </LST>
    </SECSData>
</SECSMessage>
```

#### Step 2：Host 立即回送 S1F13R（空格式 L:0 非標準，實際回傳 L:2）

```xml
<SECSMessage s="1" f="13" direction="H to E" replyBit="true"
             txid="2000530" time="2026-04-14T13:04:13.374Z" deviceID="0">
    <SECSData />   <!-- Host 發送：空 (等同 L:0 語意) -->
</SECSMessage>
<SECSMessage s="1" f="14" direction="E to H" replyBit="false"
             txid="2000530" time="2026-04-14T13:04:13.377Z" deviceID="0">
    <SECSData>
        <LST>
            <BIN>0</BIN>            <!-- COMMACK=0: 接受 -->
            <LST>
                <ASC>MXY2-200-CCD</ASC>  <!-- MDLN（Equipment 回傳） -->
                <ASC>00.01</ASC>         <!-- SOFTREV -->
            </LST>
        </LST>
    </SECSData>
</SECSMessage>
```

**分析：**

| 步驟 | txid | 方向 | 間隔 | 說明 |
|------|------|------|------|------|
| E→H S1F13R | 5 | E→H | — | Equipment 主動建立通訊，攜帶 MDLN=`MXY2-200-CCD`, SOFTREV=`00.01` |
| H→E S1F14 | 5 | H→E | 3ms | Host 回覆 COMMACK=0，空 MDLN/SOFTREV 列表 |
| H→E S1F13R | 2000530 | H→E | 16ms | Host 立即發送自身 S1F13R（空資料） |
| E→H S1F14 | 2000530 | E→H | 3ms | Equipment 回覆，**再次**回傳 MDLN/SOFTREV |

> **觀察：** txid=2000530 的 Host S1F13R 資料為空（`<SECSData />`），但 Equipment 回覆的 S1F14 中 MDLN/SOFTREV 列表有值。這是 Schmoll 設備的特殊行為：無論收到的 S1F13R 格式如何，Equipment 端的 S1F14 reply 固定回傳 MDLN/SOFTREV（使用 `L:2` 而非 `L:0`）。Host 端應能容錯接受此非標準回覆格式。
