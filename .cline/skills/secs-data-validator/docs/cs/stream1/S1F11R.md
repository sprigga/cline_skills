# S1F11R - Status Variable Namelist Request

**Direction:** Sent by Host Only

---

| **S1F11R** | Status Variable Namelist Request | Sent by Host Only |

Comment: Host sends L:0 to request all SVIDs.

Format:

```
{L:n
  SVID}
```

- **`L:n`** — 可變長度列表，包含 `n` 個 SVID 元素
- **`SVID`** — Status Variable ID（狀態變數 ID），類型 `U4:1`（varies）
- **`L:0`** — 空列表，Host 發送 `L:0` 請求所有 SVIDs

---

## TSN 結構說明

| 層級 | 變數 | 類型 | 說明 |
|------|------|------|------|
| `list0` | `L:n` | 外層列表 | n 個 SVID，`L:0` = 請求全部 |
| `list1` | `SVID` | `U4:1`（varies）| 每個狀態變數的 ID |

### `ListSplit` 解析流程

```
TSN_data = "L:3 U4:1 101 U4:1 102 U4:1 103"
                ↓
list0 = ListSplit(TSN_data)
list0[0] = "L:3"      ← 類型 token，3 個子元素
list0[1] = "U4:1 101" ← 第一個 SVID
list0[2] = "U4:1 102"
list0[3] = "U4:1 103"
                ↓
list1 = ListSplit(list0[1])
list1[0] = "U4:1"  ← 類型
list1[1] = "101"   ← 值 (SVID = "101")
```

### 解析細節

| 問題 | 說明 |
|------|------|
| 為何 `list0.Length < 1`？ | `L:0`（空列表）仍有一個 token，所以 `Length == 1` 是合法的 |
| 為何用 `Length == 2` 判斷？ | `ListSplit` 對 `"U4:1 101"` 回傳 `["U4:1", "101"]`，空值時只有一個元素 |
| 為何不驗證 `L:0`？ | 任意長度的列表都合法，Host 可送 0 到 n 個 SVID |
| `send_reply` 後做什麼？ | 應送 S1F12（Status Variable Namelist Reply），需填入 SVID+SVNAME+UNITS |

## C# Handler（dmhclient 風格）

```csharp
    // S1F11R C# Receive message - add next line to setup
    //SecsPort sp=your_SecsPort;
    sp.MessageTypeAdd( 1, 11, new SecsMsgReceiveDelegate(recv_S1F11R));

// receive method
void recv_S1F11R(object sender, int stream, int function, bool send_reply, int transID, string TSN_data, string header) {
    SecsPort sp = (SecsPort)sender;
    bool ok=true;
    while (ok) {   // break out of loop on error
        // expect L:n SVID
        // 外層列表：L:0 表示請求全部，L:n 表示指定 n 個 SVID
        string SVID;  	// U4:1 (varies)  status variable ID
        string [] list0;
        list0 = SecsPort.ListSplit(TSN_data);
        if (list0.Length < 1) { ok=false; break; }
        // 遍歷每個 SVID（index 從 1 開始，跳過 list0[0] 的類型 token）
        int n;
        for(n=1; n < list0.Length; n++) {
            string [] list1;
            list1 = SecsPort.ListSplit(list0[n]);
            if (list1.Length < 1) { ok = false; break; }
            // 有兩個元素時取值，否則為空（空 SVID）
            if (list1.Length == 2) { SVID = list1[1];} else { SVID = ""; }
            }
        if (!ok) break;
        if (send_reply) {
            string reply; // TBD create S1F12 reply with SVID+SVNAME+UNITS
            sp.SendReply(1, 12, transID, reply);
            }
        return;
        } // end while(ok)
    // bad data
    sp.SendS9(7, header);
    } // end recv_S1F11R
```

## 現代 secs4net IMessageHandler 架構

若整合進 `IMessageHandler` 架構（使用 `Item` 物件系統而非字串解析）：

```csharp
// Handlers/S1F11Handler.cs
public class S1F11Handler : IMessageHandler
{
    public int Stream => 1;
    public int Function => 11;

    public async Task HandleAsync(MessageHandlerContext wrapper)
    {
        var msg = wrapper.PrimaryMessage;
        // msg.SecsItem 是 L:n，子元素為各 SVID
        var svids = new List<uint>();
        if (msg.SecsItem is SecsItem<IEnumerable<Item>> list && list.Count > 0)
        {
            foreach (var item in list)
                svids.Add(item.GetValue<uint>());
        }
        // svids 為空 = L:0，代表請求全部

        // TODO: 查詢 SVID 對應的 SVNAME 和 UNITS，建構 S1F12 reply
        await wrapper.TryReplyAsync(null); // null → S9F7
    }
}
```

> **Note:** 在 `IMessageHandler.HandleAsync` 中，`PrimaryMessage` 由 dispatcher 擁有，handler 返回後立即 dispose，所有非同步工作必須在返回前完成。
