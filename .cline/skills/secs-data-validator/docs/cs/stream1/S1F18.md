# S1F18 - ON-LINE Acknowledge

**Direction:** Sent by Equipment Only

---

| **S1F18** | ON-LINE Acknowledge | Sent by Equipment Only |

## Format

The S1F18 message contains a single data item:

- **[ONLACK](http://hume.com/secs/items.html#ONLACK)** — Binary (B:1): Online Acknowledge status
  - `0` = OK (equipment successfully transitioned online)
  - Non-zero = Error (online transition failed; consult equipment documentation for specific error codes)

## Message Structure

```
S1F18 = ONLACK (B:1)
```

The response contains only one byte of data indicating whether the equipment acknowledged the online request.

## C# Implementation Example

The following code demonstrates how to send an S1F17 (Request Online) message and parse the S1F18 (Online Acknowledge) reply:

```csharp
// S1F18 C# Parse reply
SecsHost sp = (SecsHost)your_SecsHost;
RcResult rcr = sp.SendSecsMsg(1, 17, true, sendData, true);
string TSN_data;
bool ok = true;

while (ok) {   // break out of loop on error
    if (rcr.rc != 0) { ok = false; break; }
    TSN_data = rcr.result;
    
    // expect ONLACK
    // variables for data items and parsing
    int ONLACK;  	// B:1 (always) - online acknowledge, 0 = ok
    string[] list0;
    
    list0 = SecsPort.ListSplit(TSN_data);
    if (list0.Length < 1) { ok = false; break; }
    if (list0[0] != "B:1") { ok = false; break; }
    
    ONLACK = sp.BinToInt(list0[1]);
    
    // ONLACK value indicates success (0) or error (non-zero)
    return;  // parsed ok
} // end while(ok)
```

### Code Explanation

1. **SendSecsMsg()** — Sends S1F17 (Request Online) and waits for S1F18 reply
2. **ListSplit()** — Parses the TSN_data string into individual SECS data items
3. **BinToInt()** — Converts the binary ONLACK byte to an integer
4. **Validation** — Checks that the response contains exactly the expected `B:1` (1-byte binary) data type
