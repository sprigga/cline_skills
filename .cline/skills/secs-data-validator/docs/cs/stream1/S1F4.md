# S1F4 - Selected Equipment Status Data

**Direction:** Sent by Equipment Only

---

| **S1F4** | Selected Equipment Status Data | Sent by Equipment Only |

Comment: zero length value returned for unknown SVID

## Format

**Structure:** A list (`L`) containing `n` Status Variable (`SV`) elements

```
{L:n
  [SV]  // Repeats n times
}
```

**Element Reference:**
- `[SV](http://hume.com/secs/items.html#SV)` — Status Variable (see SECS-II item definition)

## C# Codebase Example (secs4net)

This example demonstrates how to parse S1F4 using the **secs4net** library (C# SECS-II implementation):

```csharp
// S1F4 C# Parse reply
SecsHost sp = (SecsHost)your_SecsHost;
RcResult rcr = sp.SendSecsMsg(1, 3, true, sendData, true);
string TSN_data;
bool ok = true;

while (ok) {   // break out of loop on error
    if (rcr.rc != 0) { ok = false; break; }
    TSN_data = rcr.result;
    // expect L:n SV (list of status variables)
    
    // variables for data items and parsing
    string SV;  // A:n (varies) - status variable value
    string[] list0;
    list0 = SecsPort.ListSplit(TSN_data);  // Split top-level list
    if (list0.Length < 1) { ok = false; break; }
    
    int n;
    for (n = 1; n < list0.Length; n++) {
        string[] list1;
        list1 = SecsPort.ListSplit(list0[n]);  // Parse each SV item
        if (list1.Length < 1) { ok = false; break; }
        if (list1.Length == 2) { 
            SV = list1[1];  // Extract SV value
        } else { 
            SV = ""; 
        }
    }
    
    if (!ok) break;
    return;  // parsed ok
} // end while(ok)
```

### Codebase Context

- **Library:** secs4net (C# SECS-II/HSMS-SS/GEM library by mkjeff)
- **SecsHost:** A connection object managing SECS message I/O
- **SendSecsMsg():** Sends a SECS message and waits for a reply
- **SecsPort.ListSplit():** Utility to parse SECS list structures into elements
- **RcResult:** Response object containing return code (`rc`) and result data
- **TSN_data:** The raw SECS message data string (transport-level format)
