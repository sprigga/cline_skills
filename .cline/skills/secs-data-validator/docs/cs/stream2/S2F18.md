# S2F18 - Date and Time Data

**Direction:** Host and Equipment

---

| **S2F18** | Date and Time Data | Host and Equipment |

Comment: The value should not be used for system clock synchronization.

## Format

**Message Structure:** `[TIME]` — a single TIME data element

**TIME Data Type:** ASCII string (A:32) representing date and time. The format is controlled by the ECV `TimeFormat` configuration:
- `TimeFormat = 0`: `YYMMDDHHMMSS` (12 chars) — compact 2-digit year format
- `TimeFormat = 1`: `YYYYMMDDHHMMSScc` (16 chars) — 4-digit year + centiseconds
- `TimeFormat = 2`: `YYYY-MM-DDTHH:MM:SS.s[s]*{Z|+hh:mm|-hh:mm}` (variable) — ISO 8601 with timezone

The field is allocated as A:32 (fixed 32-character ASCII), padded as needed by the equipment.

## C# Implementation Example

The following code demonstrates how to receive and parse S2F18 (Date and Time Data) from equipment in a C# application using the `secs4net` library:

```csharp
// S2F18 C# Parse Reply Handler
// sp is a SecsPort or SecsHost object reference

RcResult rcr = sp.SendSecsMsg(2, 17, true, sendData, true);
string TSN_data;
bool ok = true;

while (ok) {   // break out of loop on error
    if (rcr.rc != 0) { ok = false; break; }
    TSN_data = rcr.result;
    
    // Expect TIME data element
    string TIME;  // A:32 ASCII field (always present)
    string[] list0;
    
    // Parse the message list structure
    list0 = SecsPort.ListSplit(TSN_data);
    if (list0.Length < 1) { ok = false; break; }
    
    // Validate first element is ASCII type (prefix "A:")
    if (!list0[0].StartsWith("A:")) { ok = false; break; }
    
    // Extract the TIME value (second element if present)
    if (list0.Length == 2) { TIME = list0[1]; } else { TIME = ""; }
    
    return;  // parsed successfully
} // end while(ok)
```

### Code Explanation

- **`SendSecsMsg(2, 17, true, sendData, true)`** — Sends S2F17 request and waits for S2F18 reply
- **`SecsPort.ListSplit(TSN_data)`** — Splits the SECS message list into type-prefixed elements (e.g., `"A:32"`, `"2026-04-13T14:30:00Z"`)
- **Type validation `"A:"`** — Confirms the TIME element is ASCII-encoded
- **`TIME = list0[1]`** — Extracts the actual timestamp value from the parsed list
- **Format depends on ECV** — The actual timestamp format (compact, ISO 8601, etc.) depends on how `TimeFormat` is configured on the equipment
