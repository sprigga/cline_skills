# S2F16 - New Equipment Constant Ack

**Direction:** Equipment Only

---

| **S2F16** | New Equipment Constant Ack | Equipment Only |

Comment: if any input value is not proper, none of the input values are set

## Format 說明

S2F16 是 Equipment（設備）回覆 Host 所發送的 S2F15R（New Equipment Constant Send）之確認訊息。

| 資料項目 | 型別 | 說明 |
|----------|------|------|
| `EAC` | `B:1`（固定） | Equipment Acknowledge Code，設備確認碼；`0` 表示成功，非零表示錯誤 |

**EAC 常見值：**

| 值 | 意義 |
|----|------|
| `0` | 接受（OK） |
| `1` | 至少有一個常數 ID 不存在 |
| `2` | 至少有一個常數值超出合法範圍 |
| `3` | 至少有一個常數值型別不符 |

> 備註：若任何一筆輸入值不正確，則**所有**常數均不被設定（全部或全不原則）。

## Codebase 說明

以下 C# 代碼示範 **Host 端**在發送 S2F15R 之後，解析設備回傳的 S2F16 回覆：

```csharp
// S2F16 C# Parse reply
SecsHost sp=(SecsHost)your_SecsHost;
RcResult rcr = sp.SendSecsMsg(2, 15, true, sendData, true);
string TSN_data;
bool ok=true;
while (ok) {   // break out of loop on error
    if (rcr.rc != 0) {ok=false; break;}
    TSN_data = rcr.result;
    // expect EAC
    // variables for data items and parsing
    int EAC;    // B:1 (always)  equipment acknowledge code, 0 ok
    string [] list0;
    list0 = SecsPort.ListSplit(TSN_data);
    if (list0.Length < 1) { ok = false; break; }
    if (list0[0] != "B:1") { ok = false; break; }
    EAC = sp.BinToInt(list0[1]);
    return;  // parsed ok
    } // end while(ok)
```

| 代碼行 | 說明 |
|--------|------|
| `sp.SendSecsMsg(2, 15, true, sendData, true)` | Host 主動發送 S2F15R，並等待回覆（最後一個 `true` 表示等待回覆）；回傳 `RcResult` |
| `rcr.rc != 0` | 若 `rc != 0`，表示發送失敗（逾時、S9 錯誤等），跳出迴圈處理失敗情況 |
| `rcr.result` | 取得設備回傳的 TSN 字串資料，即 S2F16 的訊息本體 |
| `SecsPort.ListSplit(TSN_data)` | 將 TSN 字串拆解為陣列：`list0[0]` 為型別 token（應為 `"B:1"`），`list0[1]` 為值 |
| `list0[0] != "B:1"` | 驗證回覆格式是否正確（EAC 固定是 1 byte Binary） |
| `sp.BinToInt(list0[1])` | 將 Binary 字串轉換為整數，即 EAC 的數值（0 = 成功） |

**完整流程：**

```
Host                          Equipment
  |                               |
  |  S2F15R (設定常數請求)  →→→  |
  |                               | (驗證所有 ECID/ECV)
  |  ←←←  S2F16 (EAC 確認碼)    |
  |                               |
```
