**Direction:** Equipment Only

| **S2F36** | Link Event Report Acknowledge | Equipment Only |

Format:

[LRACK](http://hume.com/secs/items.html#LRACK)

        if (list0[0] != "B:1") { ok = false; break; }
        LRACK = sp.BinToInt(list0[1]);
        return;  // parsed ok
        } // end while(ok)

## Understanding Format and Codebase

### Format
The **Format** section shows a simple scalar reply:
- `[LRACK]` — a single binary item (B:1) representing the link report acknowledgment code

### Codebase (C# / secs4net)
This is a **Host-side reply parsing example** (receiving S2F36 after sending S2F35):
1. **Send request and wait**: `sp.SendSecsMsg(2, 35, true, sendData, true)` sends S2F35 and blocks for the reply
2. **Check transmission**: `rcr.rc != 0` indicates failure
3. **Parse successful reply**:
   - `SecsPort.ListSplit(TSN_data)` extracts the message
   - Validate type: `list0[0] == "B:1"`
   - `sp.BinToInt(list0[1])` converts binary to integer
4. **Use the acknowledgment**: Store LRACK for logic validation

**Reply pattern**:
- All **acknowledge messages** (S2F32, S2F34, S2F36, S2F38) follow this same simple parsing pattern
- They are binary scalars (B:1) containing a status code
- Parsing is identical: validate type, convert value, use it
- The differences are only in the message numbers and variable names
