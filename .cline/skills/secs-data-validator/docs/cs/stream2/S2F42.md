# S2F42 - Host Command Acknowledge

**Direction:** Equipment Only

---

| **S2F42** | Host Command Acknowledge | Equipment Only |

Comment: n is equal to the number of parameters having errors

Format:

{L:2
[HCACK](http://hume.com/secs/items.html#HCACK)

{L:n

{L:2

[CPNAME](http://hume.com/secs/items.html#CPNAME)

[CPACK](http://hume.com/secs/items.html#CPACK)

 }

} }

```
    // S2F42 C# Parse reply
    SecsHost sp=(SecsHost)your_SecsHost;
    RcResult rcr = sp.SendSecsMsg(2, 41, true, sendData, true);
    string TSN_data;
    bool ok=true;
    while (ok) {   // break out of loop on error
        if (rcr.rc != 0) {ok=false; break;}
        TSN_data = rcr.result;
        // expect L:2 HCACK {L:n {L:2 CPNAME CPACK}}
        // variables for data items and parsing
        int HCACK;  	// B:1 (always)  remote command acknowledge
        string CPNAME;  	// A:n (varies)  command parameter name
        int CPACK;  	// B:1 (always)  remote command parameter acknowledge, only received if error
        string [] list0;
        list0 = SecsPort.ListSplit(TSN_data);
        if (list0.Length != 3 || list0[0] != "L:2") { ok=false; break; }
        string [] list1;
        list1 = SecsPort.ListSplit(list0[1]);
        if (list1.Length < 1) { ok = false; break; }
        if (list1[0] != "B:1") { ok = false; break; }
        HCACK = sp.BinToInt(list1[1]);
        list1 = SecsPort.ListSplit(list0[2]);
        if (list1.Length < 1) { ok=false; break; }
        int n;
        for(n=1; n < list1.Length; n++) {
            string [] list2;
            list2 = SecsPort.ListSplit(list1[n]);
            if (list2.Length != 3 || list2[0] != "L:2") { ok=false; break; }
            string [] list3;
            list3 = SecsPort.ListSplit(list2[1]);
            if (list3.Length < 1) { ok = false; break; }
            if (list3.Length == 2) { CPNAME = list3[1];} else { CPNAME = ""; }
            list3 = SecsPort.ListSplit(list2[2]);
            if (list3.Length < 1) { ok = false; break; }
            if (list3[0] != "B:1") { ok = false; break; }
            CPACK = sp.BinToInt(list3[1]);
            }
        if (!ok) break;
        return;  // parsed ok
        } // end while(ok)
```

## Understanding Format and Codebase

### Format

S2F42 是 S2F41R（Host Command Send）的回覆訊息，由設備端回傳給 Host，表示遠端命令的執行結果。

#### 整體結構（三層巢狀列表）

```
{L:2                       ← 外層固定 L:2 列表
  [HCACK]                  ← 層級 1：命令整體結果碼（B:1）
  {L:n                     ← 層級 2：參數錯誤列表（n = 有錯誤的參數數量）
    {L:2                   ← 層級 3（每個錯誤項目，L:2）
      [CPNAME]             ← 有錯誤的參數名稱（A:n）
      [CPACK]              ← 該參數的錯誤碼（B:1）
    }
    ...
  }
}
```

#### 各欄位說明

| 欄位 | 類型 | 說明 |
|------|------|------|
| `HCACK` | `B:1` | Host Command Acknowledge：整體命令執行結果碼 |
| `CPNAME` | `A:n` | Command Parameter Name：發生錯誤的參數名稱 |
| `CPACK` | `B:1` | Command Parameter Acknowledge：個別參數的錯誤碼 |

#### HCACK 數值定義（GEM 標準）

| 值 | 意義 |
|----|------|
| `0` | 命令執行成功（OK） |
| `1` | 命令不存在（Command does not exist） |
| `2` | 命令已無法執行（Cannot perform now） |
| `3` | 至少有一個參數錯誤（At least one parameter error） |

#### CPACK 數值定義

| 值 | 意義 |
|----|------|
| `0` | 參數值正確（不應出現在此列表中） |
| `1` | 參數名稱不存在（CPNAME does not exist） |
| `2` | 參數值不合法（CPVAL invalid for CPNAME） |
| `3` | 參數值超出範圍（CPVAL out of range） |

#### 關鍵結構規則

1. **n 的語意**：`L:n` 的 `n` 等於**有錯誤的參數數量**（訊息標頭的 Comment 明確說明）
   - `HCACK = 0`（成功）時，`n = 0`，即 `L:0`（空列表）
   - `HCACK = 3`（參數錯誤）時，`n ≥ 1`，列表中只列出有錯的參數
2. **稀疏列表**：`L:n` 只包含**有錯誤**的參數，成功的參數不出現在此列表中
3. **類型強制**：`HCACK` 和 `CPACK` 都必須是 `B:1`（單位元組二進位），不接受其他類型

#### 對比 S2F41R（請求訊息）

| 特性 | S2F41R（請求） | S2F42（回覆） |
|------|----------------|---------------|
| 方向 | Host → Equipment | Equipment → Host |
| 第一欄位 | `RCMD`（命令名稱，A:n） | `HCACK`（結果碼，B:1） |
| 參數列表內容 | 所有參數的名稱與值 `{CPNAME, CPVAL}` | 只有錯誤參數的名稱與錯誤碼 `{CPNAME, CPACK}` |
| 參數值類型 | `CPVAL`：任意純量類型 | `CPACK`：固定 B:1 |
| 列表是否完整 | 包含全部參數 | 只包含有錯誤的參數（稀疏） |

### Format 格式範例

以下三個範例對應 S2F41R（`RCMD="PP_SELECT"`, 帶兩個參數 `PARAM1` 和 `PARAM2`）的不同回覆情境。

---

#### 範例 1：命令執行成功（HCACK=0）

> 設備成功執行命令，無任何參數錯誤，L:n 為空列表（n=0）

**TSN 字串表示：**

```
L:2
  B:1 0x00        ← HCACK = 0（OK）
  L:0             ← 無錯誤參數
```

**實際 TSN_data 字串（ListSplit 解析前）：**

```
{L:2 {B:1 \x00} {L:0}}
```

**ListSplit 解析結果：**

```
list0[0] = "L:2"
list0[1] = "{B:1 \x00}"        ← HCACK token
list0[2] = "{L:0}"             ← 空的參數錯誤列表

  list1[0] = "B:1"
  list1[1] = "\x00"            → HCACK = BinToInt → 0

  list1[0] = "L:0"             ← Length=1，迴圈不執行（n=1 不滿足 n < 1）
```

---

#### 範例 2：命令不存在（HCACK=1）

> 設備不認識此 RCMD，直接拒絕，L:n 同樣為空（n=0）

**TSN 字串表示：**

```
L:2
  B:1 0x01        ← HCACK = 1（Command does not exist）
  L:0             ← 無參數錯誤列表（整體命令失敗，不逐一回報參數）
```

**ListSplit 解析結果：**

```
list0[0] = "L:2"
list0[1] = "{B:1 \x01}"        ← HCACK token
list0[2] = "{L:0}"

  list1[0] = "B:1"
  list1[1] = "\x01"            → HCACK = 1

  list1[0] = "L:0"             ← 迴圈不執行
```

---

#### 範例 3：參數錯誤（HCACK=3，n=2）

> 命令本身合法，但兩個參數都有問題：
> - `PARAM1`：參數名稱不存在（CPACK=1）
> - `PARAM2`：參數值超出範圍（CPACK=3）

**TSN 字串表示：**

```
L:2
  B:1 0x03                    ← HCACK = 3（At least one parameter error）
  L:2                         ← 2 個參數錯誤（n=2）
    L:2
      A:6 "PARAM1"            ← CPNAME（參數名稱長度為 6）
      B:1 0x01                ← CPACK = 1（CPNAME does not exist）
    L:2
      A:6 "PARAM2"            ← CPNAME
      B:1 0x03                ← CPACK = 3（CPVAL out of range）
```

**ListSplit 解析結果（完整四層）：**

```
list0[0] = "L:2"
list0[1] = "{B:1 \x03}"
list0[2] = "{L:2 {L:2 {A:6 PARAM1} {B:1 \x01}} {L:2 {A:6 PARAM2} {B:1 \x03}}}"

  ─── 解析 HCACK ───
  list1 = ListSplit(list0[1])
  list1[0] = "B:1"
  list1[1] = "\x03"           → HCACK = 3

  ─── 解析參數錯誤列表 ───
  list1 = ListSplit(list0[2])
  list1[0] = "L:2"            ← 類型 token
  list1[1] = "{L:2 {A:6 PARAM1} {B:1 \x01}}"   ← 第 1 個錯誤項目
  list1[2] = "{L:2 {A:6 PARAM2} {B:1 \x03}}"   ← 第 2 個錯誤項目

  ─── 迴圈 n=1（PARAM1）───
  list2 = ListSplit(list1[1])
  list2[0] = "L:2"
  list2[1] = "{A:6 PARAM1}"
  list2[2] = "{B:1 \x01}"

    list3 = ListSplit(list2[1])
    list3[0] = "A:6"
    list3[1] = "PARAM1"       → CPNAME = "PARAM1"

    list3 = ListSplit(list2[2])
    list3[0] = "B:1"
    list3[1] = "\x01"         → CPACK = 1

  ─── 迴圈 n=2（PARAM2）───
  list2 = ListSplit(list1[2])
  list2[0] = "L:2"
  list2[1] = "{A:6 PARAM2}"
  list2[2] = "{B:1 \x03}"

    list3 = ListSplit(list2[1])
    list3[0] = "A:6"
    list3[1] = "PARAM2"       → CPNAME = "PARAM2"

    list3 = ListSplit(list2[2])
    list3[0] = "B:1"
    list3[1] = "\x03"         → CPACK = 3
```

---

#### 範例比較總覽

| 情境 | HCACK | L:n | 說明 |
|------|-------|-----|------|
| 命令成功 | `0` | `L:0`（空） | 全部參數正確，無錯誤列表 |
| 命令不存在 | `1` | `L:0`（空） | 命令層級失敗，不回報個別參數 |
| 無法執行 | `2` | `L:0`（空） | 時序問題，不回報個別參數 |
| 參數錯誤 | `3` | `L:n`（n≥1） | 只列出**有錯**的參數（稀疏） |

---

### Codebase（C# 解析邏輯）

此程式碼是**Host 端接收 S2F42 回覆**的解析流程（先發送 S2F41R，再解析 S2F42 回覆）：

#### 解析流程（四個層級）

```
rcr = sp.SendSecsMsg(2, 41, ...)      ← 發送 S2F41R 並等待 S2F42 回覆
│
├─ 層級 0：list0 = ListSplit(TSN_data)
│   ├─ 驗證：list0.Length == 3 且 list0[0] == "L:2"
│   ├─ list0[1] → HCACK 的原始 token
│   └─ list0[2] → L:n 的原始 token
│
├─ 層級 1a：list1 = ListSplit(list0[1])    ← 解析 HCACK
│   ├─ 驗證：list1[0] == "B:1"
│   └─ HCACK = BinToInt(list1[1])
│
├─ 層級 1b：list1 = ListSplit(list0[2])    ← 解析參數錯誤列表
│   └─ 迴圈 n = 1 .. list1.Length-1
│       │
│       ├─ 層級 2：list2 = ListSplit(list1[n])
│       │   └─ 驗證：list2.Length == 3 且 list2[0] == "L:2"
│       │
│       ├─ 層級 3a：list3 = ListSplit(list2[1])   ← 解析 CPNAME
│       │   └─ CPNAME = (list3.Length==2) ? list3[1] : ""
│       │
│       └─ 層級 3b：list3 = ListSplit(list2[2])   ← 解析 CPACK
│           ├─ 驗證：list3[0] == "B:1"
│           └─ CPACK = BinToInt(list3[1])
```

#### 關鍵解析細節

- **`list1.Length < 1` 檢查**：外層和內層都先確保 ListSplit 至少回傳一個元素（即類型 token）
- **`list1[0]` 是類型 token**：對於 L:n，`list1[0]` 是 `"L:n"`，實際子項目從 `list1[1]` 開始，所以迴圈從 `n=1` 開始
- **CPNAME 可為空**：`if (list3.Length == 2)` 處理參數名稱為空字串的情況（`list3.Length == 1` 表示沒有值）
- **CPACK 類型必須是 `B:1`**：與 CPNAME 不同，CPACK 有強制類型驗證

#### 錯誤處理

程式碼使用 `while(ok) { ... break }` 模式替代例外處理：
- 任何驗證失敗都設定 `ok=false` 並 `break`
- 最後的 `return` 只在成功解析時執行
- 此訊息為回覆訊息，**不傳送 S9F7**（S9 只針對請求訊息的解析錯誤）
