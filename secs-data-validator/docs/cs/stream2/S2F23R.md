# S2F23R - Trace Initialize Send（追蹤採樣初始化）

**方向：** Host → Equipment（Host Only）

---

| **S2F23R** | Trace Initialize Send | Host Only |

## 指令用途

S2F23R 用於**建立一個週期性的狀態變數採樣計畫**。Host 向 Equipment 指定要採樣的變數清單（SVID）、採樣頻率（DSPER）、總採樣次數（TOTSMP）以及每次回報的組大小（REPGSZ），並為此追蹤任務命名（TRID）。

計畫建立後，Equipment 不需要 Host 再次詢問，會依照設定**主動**以 S6F1R 定期推送採樣資料給 Host。

**核心語義：**「以 `DSPER` 的頻率，對指定的 `SVID` 變數採樣 `TOTSMP` 次，每 `REPGSZ` 次為一組回報，此追蹤任務命名為 `TRID`。」

**特殊操作：**
- **`TOTSMP=0`** → 終止該 `TRID` 的追蹤（等同刪除該計畫）
- **`TRID` 為空字串** → 刪除全部追蹤計畫

## 完整交易流程

```
Host  ──── S2F23R ────▶  Equipment    建立追蹤採樣計畫
Host  ◀─── S2F24  ────   Equipment    TIAACK=0 (OK) / 非0 (拒絕)
Host  ◀─── S6F1R  ────   Equipment    第 1 筆採樣資料 (DSID=1)
Host  ──── S6F2   ────▶  Equipment    確認收到
Host  ◀─── S6F1R  ────   Equipment    第 2 筆採樣資料 (DSID=2)
Host  ──── S6F2   ────▶  Equipment    確認收到
          ...（依 TOTSMP/REPGSZ 持續至結束）
```

### S2F24 回覆碼 TIAACK

| 值 | 意義 |
|----|------|
| 0 | OK / Accepted（接受） |
| 非 0 | 拒絕（Equipment 不支援、參數錯誤或資源不足）|

---

Comment: Set TOTSMP=0 to terminate a trace. Set zero-length TRID to delete all traces.

## Format

```
{L:5
[TRID](http://hume.com/secs/items.html#TRID)
[DSPER](http://hume.com/secs/items.html#DSPER)
[TOTSMP](http://hume.com/secs/items.html#TOTSMP)
[REPGSZ](http://hume.com/secs/items.html#REPGSZ)
{L:n
[SVID](http://hume.com/secs/items.html#SVID)
} }
```

## Format（格式）詳細說明

`{L:5 ... }` 表示此消息包含 **5 個元素**的 list：

- **TRID** — Trace Request ID（追蹤請求識別碼），ASCII 可變長度字串（`A:n`）
- **DSPER** — Data Sample Period（採樣週期），時間格式 `hhmmss`（固定 6 位）或 `hhmmsscc`（8 位，含百分之一秒）
- **TOTSMP** — Total Samples（總採樣數），`U4:1`（4 位元組無號整數），應為 REPGSZ 的倍數
- **REPGSZ** — Reporting Group Size（每次回報的採樣數），`U4:1`（4 位元組無號整數）
- **`{L:n SVID}`** — 嵌套 list，包含 `n` 個要採集的狀態變數 ID（`SVID`，`U4:1`）

**語義：** 「以 `DSPER` 的頻率，對指定的 `SVID` 變數採樣 `TOTSMP` 次，每 `REPGSZ` 次為一組回報。此追蹤命名為 `TRID`。」

**特殊操作：**
- **TOTSMP=0** → 終止該 TRID 的追蹤
- **TRID 為空字串** → 刪除所有追蹤

## Codebase（C# 代碼）詳細說明

### 函數參數說明

| 參數 | 含義 |
|------|------|
| `sender` | SecsPort 連接對象 |
| `stream` | SECS-II Stream 編號（這裡是 2） |
| `function` | 函數編號（這裡是 23） |
| `send_reply` | 是否需要發送回復 |
| `transID` | 交易 ID（用於配對請求和回復） |
| `TSN_data` | 傳輸的數據（SECS 格式字符串） |
| `header` | 消息頭信息 |

### 代碼邏輯分解

**1. 驗證外層 list 結構**
```csharp
list0 = SecsPort.ListSplit(TSN_data);
if (list0.Length != 6 || list0[0] != "L:5") { ok=false; break; }
```
- `ListSplit()` 將 SECS 字符串拆分成陣列
- 驗證外層有 5 個元素（`list0.Length == 6`，因為包含 `L:5` 標記本身）

**2. 逐一解析 5 個欄位**

每個欄位用 `ListSplit()` 再拆一次，取出格式標記和值：
```csharp
list1 = SecsPort.ListSplit(list0[1]);  // 拆 TRID
if (list1.Length == 2) { TRID = list1[1]; } else { TRID = ""; }
```
- `list1[0]` 是類型標記（如 `A:5`），`list1[1]` 是實際值
- `list1.Length == 2` 有值，否則設為空字串（可選欄位的處理方式）
- DSPER、TOTSMP、REPGSZ 以相同模式解析

**3. 巡訪所有 SVID（嵌套 list）**
```csharp
list1 = SecsPort.ListSplit(list0[5]);  // 拆 {L:n SVID}
for(n=1; n < list1.Length; n++) {
    list2 = SecsPort.ListSplit(list1[n]);
    if (list2.Length == 2) { SVID = list2[1]; } else { SVID = ""; }
}
```
- `n=1` 跳過 `L:n` 標記，從第一個 SVID 開始巡訪
- 每個 SVID 以相同模式取出

**4. 回復或錯誤處理**
```csharp
// 成功 → 回覆 S2F24
sp.SendReply(2, 24, transID, reply);

// 解析失敗 → 回傳 S9F7（非法資料）
sp.SendS9(7, header);
```

**5. `while(ok) { break on error }` 模式**

這是 secs4net 的慣用寫法 — 用 `while(true)` + `break` 模擬 early return，讓所有解析步驟平鋪在同一層，避免深層巢狀 `if-else`。任一步驟出錯就設 `ok=false` 並 `break`，最後由 `sendS9(7, header)` 統一處理錯誤。

### 處理流程總結

| 階段 | 操作 | 說明 |
|------|------|------|
| **解析** | `ListSplit()` | 將 SECS 字符串分解為外層 5 個元素 |
| **驗證** | 檢查 `L:5` | 確保外層 list 有 5 個欄位 |
| **取值** | 逐一 `ListSplit()` | 取出 TRID、DSPER、TOTSMP、REPGSZ |
| **巡訪** | `for` 迴圈 | 取出所有 SVID（`n` 個） |
| **回復** | `SendReply(2, 24, ...)` | 成功時發送 S2F24 |
| **錯誤** | `sendS9(7, header)` | 解析失敗時發送 S9F7 |

### S6F1 Trace Data Report 結構（KA01 Log 2026-03-18 實測）

S2F23R 成功後，Equipment 依照 REPGSZ 週期發送 S6F1。實際觀察到的 S6F1 格式如下：

```
{L:4
  DATAID  (U4:1)    — 追蹤識別碼（本次為 0，與 S2F23R 的 DATAID 一致）
  DSID    (U4:1)    — 資料集序號（第1筆=1, 第2筆=2, …）
  TIMESTAMP (A:16)  — 採樣時間戳，格式 "YYYYMMDDHHmmsscc"（如 "2026031815391731"）
  {L:5              — 5 個 SV 值（對應 SVIDs=[2004,2008,2030,2031,2032]）
    SV_2004 (A:16)  — 時間戳字串，如 "2026031815391731"
    SV_2008 (A:12)  — 設備型號，如 "MXY2-200-CCD"
    SV_2030 (U1:1)  — 數值 5
    SV_2031 (U1:1)  — 數值 4
    SV_2032 (A:4)   — 狀態字串，如 "WAIT"
  }
}
```

**第 1 筆（txid=70, T=07:39:17.313）：** DSID=1, TIMESTAMP="2026031815391731"
**第 2 筆（txid=71, T=07:39:18.314）：** DSID=2, TIMESTAMP="2026031815391831"

注意：SV 值的順序與 S2F23R 中 SVID list 的順序一一對應（2004→2008→2030→2031→2032）。

### 與 Python/dmhclient 的對應

| C# (secs4net) | Python (dmhclient) |
|---|---|
| `SecsPort.ListSplit(data)` | `lsplit(data)` |
| `sp.SendReply(s, f, tid, data)` | `sp.sendReply(s, f, tid, data)` |
| `sp.SendS9(7, header)` | `sp.sendS9(7, header)` |
| `SecsMsgReceiveDelegate` | 直接傳函式引用 |

```
    // S2F23R C# Receive message - add next line to setup
    //SecsPort sp=your_SecsPort;
    sp.MessageTypeAdd( 2, 23, new SecsMsgReceiveDelegate(recv_S2F23R));

// receive method
void recv_S2F23R(object sender, int stream, int function, bool send_reply, int transID, string TSN_data, string header) {
    SecsPort sp = (SecsPort)sender;
    bool ok=true;
    while (ok) {   // break out of loop on error
        // expect L:5 TRID DSPER TOTSMP REPGSZ {L:n SVID}
        // variables for data items and parsing
        string TRID;  	// A:n (varies)  trace request ID
        string DSPER;  	// A:6 (varies)  data sample period, hhmmss is always supported, A:8 hhmmsscc may be supported
        string TOTSMP;  	// U4:1 (varies)  total samples to be made, should be an even multiple of REPGSZ
        string REPGSZ;  	// U4:1 (varies)  reporting group size, TOTSMP modulo REPGSZ should be 0
        string SVID;  	// U4:1 (varies)  status variable ID
        string [] list0;
        list0 = SecsPort.ListSplit(TSN_data);
        if (list0.Length != 6 || list0[0] != "L:5") { ok=false; break; }
        string [] list1;
        list1 = SecsPort.ListSplit(list0[1]);
        if (list1.Length < 1) { ok = false; break; }
        if (list1.Length == 2) { TRID = list1[1];} else { TRID = ""; }
        list1 = SecsPort.ListSplit(list0[2]);
        if (list1.Length < 1) { ok = false; break; }
        if (list1.Length == 2) { DSPER = list1[1];} else { DSPER = ""; }
        list1 = SecsPort.ListSplit(list0[3]);
        if (list1.Length < 1) { ok = false; break; }
        if (list1.Length == 2) { TOTSMP = list1[1];} else { TOTSMP = ""; }
        list1 = SecsPort.ListSplit(list0[4]);
        if (list1.Length < 1) { ok = false; break; }
        if (list1.Length == 2) { REPGSZ = list1[1];} else { REPGSZ = ""; }
        list1 = SecsPort.ListSplit(list0[5]);
        if (list1.Length < 1) { ok=false; break; }
        int n;
        for(n=1; n < list1.Length; n++) {
            string [] list2;
            list2 = SecsPort.ListSplit(list1[n]);
            if (list2.Length < 1) { ok = false; break; }
            if (list2.Length == 2) { SVID = list2[1];} else { SVID = ""; }
            }
        if (!ok) break;
        if (send_reply) {
            string reply; // TBD create reply
            sp.SendReply(2, 24, transID, reply);
            }
        return;
        } // end while(ok)
    // bad data
    sp.SendS9(7, header);
    } // end recv_S2F23R
```
