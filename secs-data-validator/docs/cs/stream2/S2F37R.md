## Understanding Format and Codebase

### Format
The **Format** section shows a two-level structure:
```
{L:2
  [CEED]
  {L:n
    [CEID]
  }
}
```
- **Outer list (L:2)**: Contains CEED (enablement flag) and a list of CEIDs
- **CEID list (L:n)**: Variable-length list of collection event IDs to enable/disable

### Codebase (C# / secs4net)
The C# code implements two-level nested parsing:
1. **Level 0**: Split the outer L:2 list
2. **Extract CEED**: 
   - `list1 = SecsPort.ListSplit(list0[1])` parses the CEED item
   - Validate type: `list1[0] == "TF:1"` (true/false, 1 byte)
   - `int.Parse(list1[1])` converts to integer (0 or 1)
3. **Process CEID list**:
   - `list1 = SecsPort.ListSplit(list0[2])` parses the variable-length CEID list
   - Loop through `list1[1..n]` with counter `n`
   - For each CEID, extract the value from the parsed element
4. **Reply**: Send S2F38 acknowledgment
5. **Error handling**: Send S9F7 on parse failure

**Key observation**:
- Unlike S2F33R and S2F35R, this has only **two nesting levels**, not three
- The CEID list is a simple flat loop, not a nested L:2 structure
- This simplifies the parsing: only one nested loop instead of two

**Semantic difference**:
- `CEED = 1` means "enable all events in the CEID list"
- `CEED = 0` means "disable all events in the CEID list"
- If the CEID list is empty, the operation may apply to all events (equipment-dependent)

### CEED — Collection Event or Trace Enablement

CEED is a **True/False flag (TF:1)** that tells the equipment whether to enable or disable the specified Collection Events.

| Value | Type | Meaning |
|-------|------|---------|
| `1` | TF:1 | **Enable** — equipment begins sending S6F11 when the listed CEIDs occur |
| `0` | TF:1 | **Disable** — equipment stops sending event reports for the listed CEIDs |

CEED is a **global flag**: its effect applies to every CEID listed in the same message. It is not possible to enable some CEIDs and disable others in a single S2F37R.

**Parsing note:** `TF:1` is SECS-II's boolean type, but the C# code receives it as `int`. Using `BinToInt` and checking for non-zero (rather than comparing `== 1`) is safer, as some equipment implementations send `0xFF` for true while others send `0x01`.

### n=0 Special Case

When the CEID list is empty (`L:0`), the operation applies to **all** Collection Events:

| CEED | CEID list | Effect |
|------|-----------|--------|
| `1` | `L:0` | Enable **all** CEIDs |
| `0` | `L:0` | Disable **all** CEIDs |

The standard GEM initialization sequence uses `CEED=0, L:0` to disable all events at connection start, then selectively enables only the required CEIDs:

```
S2F37R  CEED=0, L:0  → disable all events     (clean slate)
S2F38               ← ERACK=0
S2F37R  CEED=1, L:n  → enable specific CEIDs
S2F38               ← ERACK=0
```
