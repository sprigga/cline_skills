## Understanding Format and Codebase

### Format
The **Format** section shows a three-level nested structure:
```
{L:2
  [DATAID]
  {L:a
    {L:2
      [RPTID]
      {L:b
        [VID]
      }
    }
  }
}
```
- **Outer list (L:2)**: Contains 2 elements — the DATAID and the report list
- **Report list (L:a)**: Variable-length list of reports (loop counter `a`), each containing:
  - A report ID (RPTID)
  - A variable-length list of variables (loop counter `b`)

### Codebase (C# / secs4net)
The C# code implements nested list parsing:
1. **Level 0 (top list)**: `SecsPort.ListSplit(TSN_data)` splits the outer L:2 structure
2. **Level 1 (DATAID)**: Extract from `list0[1]` (the first element inside L:2)
3. **Level 1 (report list)**: Process `list0[2]` as a variable-length list with a loop counter `a`
4. **Level 2 (each report)**: For each report in the loop:
   - `SecsPort.ListSplit(list1[a])` parses the inner L:2 structure (RPTID + variable list)
   - Extract RPTID from `list2[1]`
   - Process the variable list from `list2[2]` with a second loop counter `b`
5. **Level 3 (each variable)**: Extract VID values from the innermost list

**Key patterns**:
- **Nested loops**: Double `for` loops correspond to the `{L:a {L:b ...}}` structure in the format
- **Recursive ListSplit**: Each nesting level requires another call to `ListSplit()`
- **Index alignment**: After `ListSplit()`, index [0] is the list type code (e.g., `L:2`), and [1..n] are the elements

### VID — Variable ID 詳細說明

**VID (Variable ID)** 是 SECS-II/GEM 協議中的核心識別項，用於唯一識別設備上的**資料值變數（Data Value Variable, DVVAL）**。

#### 定義

| 屬性 | 值 |
|------|-----|
| **全名** | Variable ID |
| **SECS 資料型態** | `A:n (varies)` — ASCII 字串，長度可變 |
| **識別對象** | DVVAL（Data Value Variable）— 設備上定義的資料值變數 |
| **命名規則** | 由設備廠商自行定義，通常為有意義的字串（如 `"Temperature"`、`"Pressure"`、`"WaferCount"`） |

#### VID 與 SVID 的區別

SECS-II 中有兩種常見的變數識別碼，容易混淆：

| 比較項 | **VID** (Variable ID) | **SVID** (Status Variable ID) |
|--------|----------------------|-------------------------------|
| **資料型態** | `A:n`（ASCII 字串） | `U4:1`（4 位元組無號整數） |
| **識別對象** | DVVAL（Data Value Variable） | SV（Status Variable） |
| **用途** | 報告定義 (S2F33)、事件連結、資料收集 | 狀態查詢 (S1F3/S1F4)、即時狀態回報 |
| **命名方式** | 字串名稱，由廠商定義 | 數字 ID，由廠商定義 |
| **對應訊息** | S1F21R/S1F22（查詢名稱清單）、S2F33/S2F34（報告定義） | S1F3R/S1F4（狀態查詢） |

#### 在 S2F33R 中的角色

在 S2F33R（Define Report）中，VID 出現在最內層的 `{L:b VID}` 結構中，定義每個報告（RPTID）包含哪些變數：

```
{L:b VID}   ← 一個報告包含 b 個 VID
```

**語義**：「報告 RPTID 應包含 VID-1、VID-2、...、VID-b 這 b 個變數的值。」

當設備觸發收集事件（CEID）並發送 S6F11R（Event Report）時，會按照 S2F33R 定義的 VID 順序，依序回傳對應的變數值（`V`），形成 `{L:b V}` 結構。

#### 實際範例

假設設備定義了以下 DVVAL：

| VID | 說明 | 回傳值範例 |
|-----|------|-----------|
| `"Temperature"` | 腔體溫度 | `"235.5"` |
| `"Pressure"` | 腔體壓力 | `"1.02"` |
| `"WaferCount"` | 晶圓計數 | `"25"` |
| `"FlowRate"` | 氣體流量 | `"50.3"` |

Host 發送 S2F33R 定義兩個報告：

```
// RPTID=1（製程報告）：包含溫度、壓力、晶圓數
// RPTID=2（計量報告）：包含氣體流量

S2F33R W
<L:2
  <U4:1>                     // DATAID = 1
  <L:2                       // a = 2 個報告定義
    <L:2
      <U4:1>                 // RPTID = 1
      <L:3                   // b = 3 個 VID
        <A:11 "Temperature">
        <A:8 "Pressure">
        <A:10 "WaferCount">
      >
    >
    <L:2
      <U4:2>                 // RPTID = 2
      <L:1                   // b = 1 個 VID
        <A:8 "FlowRate">
      >
    >
  >
>
```

當 CEID 觸發 RPTID=1 時，設備回傳 S6F11R：

```
S6F11R
<L:4
  <U4:1>                     // DATAID
  <U4:100>                   // CEID
  <L:2                       // 2 個報告
    <L:2
      <U4:1>                 // RPTID = 1
      <L:3                   // 3 個值（對應 3 個 VID）
        <A:5 "235.5">       // Temperature 的值
        <A:4 "1.02">        // Pressure 的值
        <A:2 "25">          // WaferCount 的值
      >
    >
    ...
  >
>
```

#### 相關訊息

| 訊息 | 說明 |
|------|------|
| **S1F21R / S1F22** | 查詢設備支援的 VID 清單及其名稱（DVVALNAME）和單位（UNITS） |
| **S1F23R / S1F24** | 查詢收集事件（CEID）關聯的 VID 清單 |
| **S2F33R / S2F34** | 定義報告：指定 RPTID 包含哪些 VID |
| **S2F35R / S2F36** | 連結事件：將 CEID 與 RPTID 關聯 |
| **S2F45R / S2F46** | 定義 VID 的限制屬性（上下限、警戒值） |
| **S2F47R / S2F48** | 查詢 VID 的限制屬性 |
| **S6F11R** | 事件報告：依 RPTID 定義的 VID 順序回傳值 |

### Error Handling
If validation fails at any nesting level, `ok = false` triggers the loop exit and sends `S9F7` (illegal data).
