# S2F24 - Trace Initialize Acknowledge（追蹤採樣初始化確認）

**方向：** Equipment → Host（Equipment Only）

---

| **S2F24** | Trace Initialize Acknowledge | Equipment Only |

## 指令用途

S2F24 是 **S2F23R 的配對回覆**。Equipment 收到 Host 的追蹤採樣計畫後，以此訊息回報是否接受。

訊息格式極簡：**只有一個 `B:1` 欄位（TIAACK）**，其值決定後續行為：

| TIAACK 值 | 意義 | Host 應對 |
|-----------|------|----------|
| `0x00` | Acknowledge（接受） | 等待 Equipment 定期發送 S6F1R |
| `0x01` | Refuse — 該 TRID 的追蹤已在執行中 | 先發 `TOTSMP=0` 終止舊追蹤，再重新送 S2F23R |
| `0x02` | Refuse — Equipment 空間不足 | 減少 TOTSMP 或 SVID 數量後重試 |

## Format

The S2F24 message contains a single data item:

- **TIAACK** (Trace Initialize Acknowledge): A binary value (B:1, always 1 byte) representing the trace initialization acknowledgement code. This indicates whether the Equipment successfully initialized tracing.
  - `0x00` = Acknowledge (success)
  - `0x01` = Refuse, already performing a requested trace
  - `0x02` = Refuse, insufficient space

For detailed specifications, see: [TIAACK](http://hume.com/secs/items.html#TIAACK)

## Format（格式）詳細說明

```
B:1  TIAACK
```

- **TIAACK** — Trace Initialize Acknowledge Code，`B:1`（1 位元組 Binary）
  - `0x00` = 接受，Equipment 已建立追蹤計畫，將依照 DSPER 週期發送 S6F1R
  - `0x01` = 拒絕，該 TRID 的追蹤已在執行中（重複初始化）
  - `0x02` = 拒絕，Equipment 空間不足（超出支援的追蹤數量或 SVID 數量）

## C# Code Example - Parsing S2F24 Reply

```csharp
// S2F24 C# Parse reply
SecsHost sp = (SecsHost)your_SecsHost;
RcResult rcr = sp.SendSecsMsg(2, 23, true, sendData, true);
string TSN_data;
bool ok = true;

while (ok) {   // break out of loop on error
    if (rcr.rc != 0) { ok = false; break; }
    
    // Get the raw TSN (Transaction/Sequence Number) data from the reply
    TSN_data = rcr.result;
    
    // expect TIAACK (Trace Initialize Acknowledge)
    // variables for data items and parsing
    int TIAACK;  	// B:1 (always)  trace acknowledgement code
    string[] list0;
    
    // Split the TSN data into individual elements
    list0 = SecsPort.ListSplit(TSN_data);
    
    // Validate that we have at least one element
    if (list0.Length < 1) { ok = false; break; }
    
    // Validate that the first element is a binary type with 1 byte (B:1)
    if (list0[0] != "B:1") { ok = false; break; }
    
    // Convert the binary data (second element) to an integer
    TIAACK = sp.BinToInt(list0[1]);
    
    return;  // parsed ok
} // end while(ok)
```

### Code Explanation

| Component | Meaning |
|-----------|---------|
| `SendSecsMsg(2, 23, true, sendData, true)` | Sends S2F23 (Trace Initialize) and waits for S2F24 reply. Parameters: Stream 2, Function 23, expect reply, data, wait flag |
| `rcr.rc` | Return code from the send operation (0 = success) |
| `rcr.result` | The TSN data string containing the reply message structure |
| `SecsPort.ListSplit(TSN_data)` | Parses the TSN data string into an array of formatted elements (e.g., `["B:1", "0x00"]`) |
| `list0[0]` | First element specifying data type and length (e.g., "B:1" = Binary, 1 byte) |
| `list0[1]` | The actual binary data value |
| `sp.BinToInt(list0[1])` | Converts binary data to integer for easier comparison |

## 代碼邏輯說明（繁體中文）

**1. 發送 S2F23R 並等待 S2F24 回覆**
```csharp
RcResult rcr = sp.SendSecsMsg(2, 23, true, sendData, true);
if (rcr.rc != 0) { /* 傳輸失敗，無回覆 */ }
```
- `rcr.rc != 0` 表示網路層錯誤或逾時，非 TIAACK 拒絕——兩者要分開處理。

**2. 解析單一欄位**
```csharp
list0 = SecsPort.ListSplit(TSN_data);
if (list0[0] != "B:1") { ok = false; break; }
TIAACK = sp.BinToInt(list0[1]);
```
- S2F24 沒有外層 `L:n`，`list0[0]` 直接是型別標記 `B:1`，`list0[1]` 是值。
- 與 S2F23R 的多層嵌套解析不同，這裡只需一次 `ListSplit()`。

**3. `while(ok) { break }` 錯誤處理模式**

這是 secs4net 的慣用寫法（詳見 S2F23R 說明）。任一驗證失敗就 `ok=false; break`，流程平鋪無巢狀，易於閱讀。

**處理流程總結**

| 階段 | 操作 | 說明 |
|------|------|------|
| **發送** | `SendSecsMsg(2, 23, ...)` | 送出 S2F23R 並阻塞等待 S2F24 |
| **驗證傳輸** | `rcr.rc != 0` | 檢查是否收到回覆 |
| **解析** | `ListSplit(TSN_data)` | 取出 `B:1` 型別標記與值 |
| **驗證型別** | `list0[0] != "B:1"` | 確認回覆格式正確 |
| **取值** | `sp.BinToInt(list0[1])` | 將 Binary 轉為整數 TIAACK |
