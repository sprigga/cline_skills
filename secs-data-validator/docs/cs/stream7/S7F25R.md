# S7F25R - Formatted Process Program Request

**Direction:** Sent by Host and Equipment

---

| **S7F25R** | Formatted Process Program Request | Sent by Host and Equipment |

### 用意與目的

S7F25R（**Formatted Process Program Request**）是 SECS-II Stream 7（製程程式管理）中的**配方下載請求訊息**。發送方用一個 PPID 向對方請求取回對應的格式化製程程式完整內容，對方以 S7F26 回覆。

#### 在 GEM 工作流程中的定位

```
Host                                Equipment
  │                                     │
  │── S7F25R (PPID="RECIPE_GOLD") ────▶│  ← 本訊息：請求取回配方
  │◀──────────── S7F26 (L:4 ...) ──────│  設備回傳完整格式化配方
  │                                     │
  │  Host 收到後可做比對/備份/稽核        │
```

S7F25R 屬於 **SEMI GEM（E30）規範 Required GEM Capability #7：Process Program Management** 的組成部分。典型使用情境：

- **備份/稽核：** Host 定期從設備拉回現有配方，確認與 MES 資料庫一致。
- **配方驗證：** Host 上傳配方（S7F23R）後，再用 S7F25R 讀回做 round-trip 驗證。
- **設備轉移：** 從舊設備讀取配方，傳送至新設備。

#### 訊息名稱解析

| 部分 | 含義 |
|------|------|
| `S7` | Stream 7 — 製程程式管理 |
| `F25` | Function 25 — 第 25 號功能（Request） |
| `R` | Reply requested（需要回覆）— 發送方期待收到 S7F26 作為資料回應 |

> **為何需要回覆（R）？** 這是一個**同步資料請求**：發送方送出 PPID，接收方必須在 T3 逾時前回傳 S7F26（含完整格式化配方資料）。若接收方無此 PPID 或無法提供，應回覆空 S7F26（PPID 為空字串，命令清單為 L:0）或由通訊層回應 S9F7（格式錯誤）。

#### 與相關訊息的關係

| 訊息 | 方向 | 用途 |
|------|------|------|
| **S7F25R** | Host ↔ Equipment | 請求取回格式化配方（本文件） |
| S7F26 | Equipment/Host → 對方 | 回傳完整格式化配方（L:4 結構） |
| S7F23R | Host → Equipment | 上傳格式化配方至設備 |
| S7F24 | Equipment → Host | 確認/拒絕 S7F23R 接收（ACKC7） |
| S7F5R | Host ↔ Equipment | 請求取回非格式化（純文字）配方 |
| S7F6 | Equipment/Host → 對方 | 回傳非格式化配方內容 |
| S7F17R | Host → Equipment | 刪除指定配方 |
| S7F19R | Host → Equipment | 查詢設備上的配方清單 |

### Format (TSN)

```
[PPID]
```

**Format Explanation:**

這是一個**最簡單的單項目訊息**，只有一個資料項目：

- **`[PPID]`** — Process Program ID（製程程式 ID），型別 `A:80`（ASCII 字串，最大 80 字元）。指定要請求的格式化製程程式名稱。

> **注意：** PPID 可以是空字串（`A:0`），在某些設備實作中表示「請求預設配方」或「請求當前正在執行的配方」。具體語意由設備規格書（ECS）定義。

### 資料型別說明與範例

S7F25R 只有一個資料項目 PPID，型別固定為 `A`（ASCII 字串）。

#### 型別速查表

| 欄位 | SECS 型別 | 長度限制 | 說明 |
|------|-----------|----------|------|
| PPID | `A` (ASCII) | 最大 80 字元 | 製程程式 ID，唯一識別配方名稱 |

#### PPID 型別：`A` (ASCII 字串)

PPID 是**唯一**的資料項目，型別固定為 ASCII。命名規則通常由工廠或 MES 系統定義。

**secs4net 建立 S7F25R：**

```csharp
// 請求名稱為 "RECIPE_GOLD" 的格式化配方
var msg = new SecsMessage(7, 25, replyExpected: true,
    Item.A("RECIPE_GOLD"));
```

**SML 表示法：**

```
S7F25R
'RECIPE_GOLD' A
.
```

**常見 PPID 命名範例：**

| 場景 | PPID 值 | 說明 |
|------|---------|------|
| 生產配方 | `"RECIPE_HOT_5NM"` | 熱製程，5nm 節點 |
| 測試配方 | `"TEST_DRY_RUN"` | 設備自我檢測用 |
| 預設配方 | `"DEFAULT"` | 未指定時的基準配方 |
| 空字串請求 | `""` | 部分設備解釋為「目前配方」 |
| 數字 ID | `"PP001"` | 短代碼風格，MES 常見 |

**secs4net 讀取收到的 S7F25R（設備端 handler）：**

```csharp
public async Task HandleAsync(PrimaryMessageWrapper wrapper)
{
    var msg = wrapper.PrimaryMessage;
    // S7F25R 的 body 就是 PPID 字串（Item.A 型別）
    string ppid = msg.SecsItem?.GetValue<string>() ?? string.Empty;

    // 查詢本地配方儲存
    var recipe = _recipeStore.GetFormatted(ppid);
    if (recipe == null)
    {
        // 配方不存在，回覆空 S7F26
        await wrapper.TryReplyAsync(BuildEmptyS7F26());
        return;
    }
    await wrapper.TryReplyAsync(BuildS7F26(recipe));
}

private static SecsMessage BuildEmptyS7F26() =>
    new SecsMessage(7, 26,
        Item.L(
            Item.A(""),          // PPID 空字串表示查無此配方
            Item.A(""),          // MDLN
            Item.A(""),          // SOFTREV
            Item.L()             // 空命令清單
        ));
```

#### 完整互動範例：Host 請求並解析回覆

```csharp
// Host 端：發送 S7F25R 並等待 S7F26
public async Task<FormattedRecipe?> RequestFormattedRecipeAsync(
    ISecsGem secsGem, string ppid, CancellationToken ct = default)
{
    var request = new SecsMessage(7, 25, replyExpected: true,
        Item.A(ppid));

    SecsMessage reply = await secsGem.SendAsync(request, ct);

    // S7F26 結構：L:4 { PPID, MDLN, SOFTREV, L:c }
    var body = reply.SecsItem;
    if (body is null || body.Count < 4) return null;

    string replyPpid    = body[0].GetValue<string>();
    string mdln         = body[1].GetValue<string>();
    string softrev      = body[2].GetValue<string>();

    // 空 PPID 回覆 = 設備表示查無此配方
    if (string.IsNullOrEmpty(replyPpid)) return null;

    var commands = new List<(string CCode, List<string> Params)>();
    var cmdList = body[3];   // L:c
    for (int i = 0; i < cmdList.Count; i++)
    {
        var cmd  = cmdList[i];           // L:2
        string ccode  = cmd[0].GetValue<string>();
        var parmList  = cmd[1];          // L:p

        var parms = new List<string>();
        for (int j = 0; j < parmList.Count; j++)
            parms.Add(parmList[j].GetValue<string>());

        commands.Add((ccode, parms));
    }

    return new FormattedRecipe(replyPpid, mdln, softrev, commands);
}
```

### 通訊流程

```
Host                                Equipment
  │                                     │
  │─── S7F25R ─────────────────────────▶│
  │    Item.A("RECIPE_GOLD")            │  設備查詢本地配方儲存
  │                                     │
  │◀────────────── S7F26 ───────────────│
  │    L:4 {                            │
  │      A "RECIPE_GOLD"                │
  │      A "EQ-MODEL-1"                 │
  │      A "1.0.0"                      │
  │      L:2 {                          │
  │        L:2 { A "HEAT"  L:1 {F4 350.0} }
  │        L:2 { A "PURGE" L:1 {U4 60}  }
  │      }                              │
  │    }                                │
```

> 若 PPID 不存在，設備可回覆空 S7F26（`L:4 { A"" A"" A"" L:0 }`）或不回覆（視設備實作而定）。

### C# secs4net 實作

#### 設備端：接收 S7F25R 並回覆 S7F26

```csharp
// Handlers/S7F25Handler.cs
using Secs4Net;

public sealed class S7F25Handler : IMessageHandler
{
    public int Stream   => 7;
    public int Function => 25;

    private readonly IRecipeStore _recipeStore;

    public S7F25Handler(IRecipeStore recipeStore)
        => _recipeStore = recipeStore;

    public async Task HandleAsync(PrimaryMessageWrapper wrapper)
    {
        var msg  = wrapper.PrimaryMessage;
        string ppid = msg.SecsItem?.GetValue<string>() ?? string.Empty;

        var recipe = _recipeStore.GetFormatted(ppid);
        var reply  = recipe != null
            ? BuildS7F26(recipe)
            : BuildEmptyS7F26(ppid);

        await wrapper.TryReplyAsync(reply);
    }

    private static SecsMessage BuildS7F26(FormattedRecipe r)
    {
        var cmdItems = r.Commands
            .Select(c => Item.L(
                Item.A(c.CCode),
                Item.L(c.Params.Select(p => Item.A(p)).ToArray())
            ))
            .ToArray();

        return new SecsMessage(7, 26,
            Item.L(
                Item.A(r.Ppid),
                Item.A(r.Mdln),
                Item.A(r.SoftRev),
                Item.L(cmdItems)
            ));
    }

    private static SecsMessage BuildEmptyS7F26(string ppid) =>
        new SecsMessage(7, 26,
            Item.L(
                Item.A(""),   // 空 PPID 表示查無此配方
                Item.A(""),
                Item.A(""),
                Item.L()
            ));
}
```

#### 在 Program.cs 中註冊

```csharp
factory.Register(new S7F25Handler(recipeStore));
```

### 單元測試

```csharp
public sealed class S7F25HandlerTests
{
    [Fact]
    public async Task ReturnsS7F26_WhenPpidExists()
    {
        var store   = new FakeRecipeStore();
        var handler = new S7F25Handler(store);
        var request = new SecsMessage(7, 25, replyExpected: true,
            Item.A("RECIPE_GOLD"));

        var (replied, replyMsg) = await SimulateHandleAsync(handler, request);

        Assert.True(replied);
        Assert.Equal(7,  replyMsg!.S);
        Assert.Equal(26, replyMsg.F);
        Assert.Equal("RECIPE_GOLD", replyMsg.SecsItem![0].GetValue<string>());
    }

    [Fact]
    public async Task ReturnsEmptyS7F26_WhenPpidNotFound()
    {
        var store   = new FakeRecipeStore();
        var handler = new S7F25Handler(store);
        var request = new SecsMessage(7, 25, replyExpected: true,
            Item.A("NO_SUCH_RECIPE"));

        var (replied, replyMsg) = await SimulateHandleAsync(handler, request);

        Assert.True(replied);
        Assert.Equal("", replyMsg!.SecsItem![0].GetValue<string>());
        Assert.Equal(0, replyMsg.SecsItem[3].Count); // 空命令清單
    }

    [Fact]
    public async Task HandlesEmptyPpid()
    {
        var handler = new S7F25Handler(new FakeRecipeStore());
        var request = new SecsMessage(7, 25, replyExpected: true,
            Item.A(""));

        var (replied, _) = await SimulateHandleAsync(handler, request);
        Assert.True(replied);
    }
}
```

### 與 S7F5R 的比較

| 特性 | S7F25R / S7F26 | S7F5R / S7F6 |
|------|----------------|--------------|
| 配方格式 | **格式化**（L:4 巢狀結構，含 CCODE/PPARM） | **非格式化**（純文字 ASCII body） |
| 適用場景 | 需要解析命令與參數的自動化流程 | 需要人類可讀或逐行處理的簡單腳本 |
| 回覆複雜度 | S7F26 需解析 L:4 → L:c → L:2 → L:p | S7F6 直接讀取字串 |
| GEM 必要性 | GEM Capability #7（必要） | 選用，視設備規格 |
