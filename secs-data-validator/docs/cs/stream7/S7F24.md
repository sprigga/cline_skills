# S7F24 - Formatted Process Program Acknowledge

**Direction:** Sent by Host and Equipment

---

| **S7F24** | Formatted Process Program Acknowledge | Sent by Host and Equipment |

Comment: Reply to S7F23R. Contains the S7 acknowledge code for the formatted process program send.

### 格式（Format）

```
[ACKC7]
```

**格式說明：**
- **`[ACKC7]`** — S7 Acknowledge Code，型別為 `B:1`（Binary，固定 1 byte）。用來回覆 S7F23R（Formatted Process Program Send）的處理結果。

---

### 用途

S7F24 是 **S7F23R（Formatted Process Program Send）的強制回覆訊息**。

當設備或 Host 收到 S7F23R（含有製程程式 PPID、MDLN、SOFTREV 和命令清單）後，必須立即回覆 S7F24 來告知對方「這個製程程式收到了，結果如何」。

**訊息流向：** 雙向（Host ↔ Equipment），誰收到 S7F23R 就由誰發送 S7F24。

S7F24 **必須在 T3 逾時（本專案設定 45 秒）內回覆**，否則發送方會觸發通訊錯誤。

---

### ACKC7 值定義（SEMI E5 標準）

| ACKC7 值 | 常數名稱（本專案） | 說明 |
|----------|--------------------|------|
| `0x00` | `Ackc7Accepted` | 已接受（Accepted） |
| `0x01` | `Ackc7CmdNotSupported` | 不支援此命令（Command not supported） |
| `0x02` | `Ackc7LengthError` | 格式或長度錯誤（Length / format error） |
| `0x03` | `Ackc7AlreadyExists` | 製程程式已存在（Already exists） |
| `0x04` | `Ackc7InsufficientSpace` | 儲存空間不足（Insufficient space） |
| `0x06` | `Ackc7InUse` | 製程程式使用中（In use） |

> 注意：`0x05` 在 SEMI E5 標準中未定義（保留）。

---

### 通訊流程

```
Host / Equipment              接收方
       |── S7F23R ──────────────▶|   傳送格式化製程程式（L:4 結構）
       |                          |   解析並驗證：PPID、MDLN、SOFTREV、命令清單
       |◀─────────── S7F24 ───────|   回覆 ACKC7 (B:1)
```

---

### C# secs4net 實作

S7F24 **由 `S7F23Handler` 產生並發送**，不是獨立的 Handler，而是在解析完 S7F23R 後，由 `HandleAsync` 自動建立並回覆。

#### 檔案位置

- **發送端（接收 S7F23R 並回覆 S7F24）：** `SecsApp/Handlers/S7F23Handler.cs`

#### S7F24 發送邏輯（摘自 S7F23Handler.cs）

```csharp
// SecsApp/Handlers/S7F23Handler.cs
public async Task HandleAsync(PrimaryMessageWrapper wrapper, CancellationToken ct)
{
    // Parse() 驗證 S7F23R 格式，回傳對應的 ACKC7 byte
    byte ackc7 = Parse(wrapper.PrimaryMessage.SecsItem);

    using var s7f24 = new SecsMessage(7, 24)
    {
        Name     = "FormattedProcessProgramAck",
        SecsItem = B(ackc7),   // B:1 — 單一 byte 回覆碼
    };
    bool sent = await wrapper.TryReplyAsync(s7f24, ct);
    Console.WriteLine($"[TX] S7F24 ACKC7=0x{ackc7:X2} → {(sent ? "OK" : "Already replied")}");
}
```

`Item.B(byte value)` 建立一個 `Binary:1` 格式的 `Item`，對應 ACKC7 的 `B:1` 型別規格。

#### ACKC7 常數定義（摘自 S7F23Handler.cs）

```csharp
public const byte Ackc7Accepted          = 0x00;
public const byte Ackc7CmdNotSupported   = 0x01;
public const byte Ackc7LengthError       = 0x02;
public const byte Ackc7AlreadyExists     = 0x03;
public const byte Ackc7InsufficientSpace = 0x04;
public const byte Ackc7InUse            = 0x06;
```

### Host 端發送 S7F23R 並接收 S7F24（OutboundCommands.cs）

`SendS7F23AndWaitS7F24Async()` 已實作於 `SecsApp/OutboundCommands.cs`，封裝了完整的發送流程與 S7F24 解析。

#### 方法簽章

```csharp
internal static async Task<byte> SendS7F23AndWaitS7F24Async(
    ISecsGem secsGem,
    string ppid,
    string mdln,
    string softrev,
    IEnumerable<(string ccode, IEnumerable<Item> pparms)> commands,
    CancellationToken ct)
```

| 參數 | 型別 | 說明 |
|------|------|------|
| `ppid` | `string` | 製程程式 ID（A:80，允許空字串） |
| `mdln` | `string` | 設備型號（A:20） |
| `softrev` | `string` | 軟體版本（A:20） |
| `commands` | `IEnumerable<(string, IEnumerable<Item>)>` | 命令清單，PPARM 可為任意 secs4net `Item` 型別 |
| 回傳 | `Task<byte>` | ACKC7 byte（`0x00` = 成功） |

#### 實作（摘自 OutboundCommands.cs）

```csharp
// SecsApp/OutboundCommands.cs
var cmdItems = commands
    .Select(cmd => L(A(cmd.ccode), L(cmd.pparms.ToArray())))
    .ToArray();

using var s7f23 = new SecsMessage(7, 23)
{
    Name     = "FormattedProcessProgramSend",
    SecsItem = L(A(ppid), A(mdln), A(softrev), L(cmdItems)),
};

Console.WriteLine($"[TX] S7F23R PPID=\"{ppid}\" MDLN=\"{mdln}\" SOFTREV=\"{softrev}\" commands={cmdItems.Length}");
using var s7f24 = await secsGem.SendAsync(s7f23, ct);

var root = s7f24.SecsItem;
if (root is null || root.Format != SecsFormat.Binary || root.Count != 1)
{
    Console.WriteLine("[ERROR] S7F24: expected ACKC7 (B:1)");
    return S7F23Handler.Ackc7LengthError;
}

byte ackc7 = root.FirstValue<byte>();
Console.WriteLine($"[INFO] S7F24 ACKC7=0x{ackc7:X2} ({Ackc7Description(ackc7)})");
return ackc7;
```

> `root.FirstValue<byte>()` 是 secs4net 2.x 讀取 `B:1` 底層值的慣用寫法，等同於 `root[0]` 但型別安全、不需手動索引。

#### ACKC7 描述（私有 helper）

```csharp
private static string Ackc7Description(byte ackc7) => ackc7 switch
{
    0x00 => "Accepted",
    0x01 => "Command not supported",
    0x02 => "Length / format error",
    0x03 => "Already exists",
    0x04 => "Insufficient space",
    0x06 => "In use",
    _    => $"Unknown (0x{ackc7:X2})",
};
```

#### 在 Program.cs 中的呼叫範例

```csharp
// SecsApp/Program.cs（startup sequence 中呼叫）
byte ackc7 = await OutboundCommands.SendS7F23AndWaitS7F24Async(
    secsGem,
    ppid:    "RECIPE_HOT_01",
    mdln:    "EtchSystem-A",
    softrev: "v1.0.0",
    commands:
    [
        ("TEMP_SET",  [F4(300.0f)]),
        ("PRESS_SET", [F4(5.0f)]),
    ],
    ct: cts.Token);

if (ackc7 == S7F23Handler.Ackc7Accepted)
    Console.WriteLine("[INFO] Process program uploaded successfully.");
else
    Console.WriteLine($"[WARN] Upload rejected: ACKC7=0x{ackc7:X2}");
```
