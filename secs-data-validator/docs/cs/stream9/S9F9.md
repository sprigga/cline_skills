# S9F9 - Transaction Timeout

**Direction:** Sent by Equipment Only

---

## Format

```
SHEAD  B:10  (always)
```

**`SHEAD`**（Sent Header）：
- 資料型態：`B:10`（Binary，固定 10 bytes）
- 內容：**Host 先前送出、但設備未在 T3 時限內收到回覆的原始訊息 header**
- 與 S9F1/F3/F5/F7 的 `MHEAD`（received header）不同，`SHEAD` 指的是設備端**已發送**給 Host 的訊息 header
- Header 欄位結構（10 bytes）：

| Byte | 內容 |
|------|------|
| 0–1  | Device ID（含 R-bit） |
| 2    | Stream number（含 reply bit） |
| 3    | Function number |
| 4–5  | Block 計數 |
| 6–9  | System bytes（Transaction ID） |

> **注意：** S9F9 表示 **T3 Transaction Timeout**——設備送出需要回覆的訊息後，超過 T3 時間仍未收到 Host 的 reply，因此以 S9F9 通知 Host。Host 端收到後應確認是否有對應的 transaction 尚未處理。

---

- `sender` — 觸發事件的 `SecsHost` 物件，轉型後使用
- `while (ok)` — 用 break 取代巢狀 if，讓解析失敗時可快速跳出

```csharp
        int [] SHEAD;  	// B:10 (always)  message header of sent block
        string [] list0;
        list0 = SecsPort.ListSplit(TSN_data);
```
- 變數名稱為 `SHEAD`（Sent Header）而非 `MHEAD`（Received Header），強調這是設備**送出**的訊息 header
- `SecsPort.ListSplit(TSN_data)` — 將 TSN 字串分割為 token 陣列；`list0[0]` 為型態 token（如 `"B:10"`），`list0[1]`～`list0[10]` 為各 byte 的 hex 字串

```csharp
        if (list0.Length < 1) { ok = false; break; }
        if (!list0[0].StartsWith("B:")) { ok = false; break; }
```
- 驗證資料存在且型態為 Binary（`B:`），否則視為解析失敗

```csharp

```
- `list0.Length - 1` — 排除第一個 token（型態），其餘為 byte 值
- `sp.BinToInt(...)` — 將 Binary hex token 轉為 int，存入 `SHEAD[]` 陣列
- 可由 `SHEAD[6]`～`SHEAD[9]`（System bytes）取得 Transaction ID，比對哪個 transaction 超時
