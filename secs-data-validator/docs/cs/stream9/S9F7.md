# S9F7 - Illegal Data

**Direction:** Sent by Equipment Only

---

## Format

```
MHEAD  B:10  (always)
```

**`MHEAD`**（Message Header）：
- 資料型態：`B:10`（Binary，固定 10 bytes）
- 內容：**Host 傳給設備的原始訊息 header**，設備因訊息資料內容不合法（格式錯誤、值域錯誤等）而以 S9F7 回報錯誤
- Header 欄位結構（10 bytes）：

| Byte | 內容 |
|------|------|
| 0–1  | Device ID（含 R-bit） |
| 2    | Stream number（含 reply bit） |
| 3    | Function number |
| 4–5  | Block 計數 |
| 6–9  | System bytes（Transaction ID） |

> **注意：** S9F7 是 SECS-II 規格中最常見的錯誤回覆。Host 送出任何格式有誤的訊息時，設備應回傳 S9F7。在 C# cslib 中，當 Host 端的 handler 解析失敗時，通常也會呼叫 `sp.SendS9(7, header)` 主動發出此訊息。

---

## Codebase 說明

```csharp
    // S9F7 C# Receive message - add next line to setup
    //SecsHost sp=your_SecsHost;
    sp.MessageTypeAdd( 9, 7, new SecsMsgReceiveDelegate(recv_S9F7));
```
- `sp.MessageTypeAdd(9, 7, ...)` — 向 `SecsHost` 註冊 Stream 9, Function 7 的接收處理器
- `SecsMsgReceiveDelegate` — cslib 定義的訊息接收委派型別

```csharp
void recv_S9F7(object sender, int stream, int function, bool send_reply, int transID, string TSN_data, string header) {
    SecsHost sp = (SecsHost)sender;
    bool ok=true;
    while (ok) {   // break out of loop on error
```
- `sender` — 觸發事件的 `SecsHost` 物件，轉型後使用
- `while (ok)` — 用 break 取代巢狀 if，讓解析失敗時可快速跳出

```csharp
        int [] MHEAD;  	// B:10 (always)  message header of received block
        string [] list0;
        list0 = SecsPort.ListSplit(TSN_data);
```
- `SecsPort.ListSplit(TSN_data)` — 將 TSN 字串分割為 token 陣列；`list0[0]` 為型態 token（如 `"B:10"`），`list0[1]`～`list0[10]` 為各 byte 的 hex 字串

```csharp
        if (list0.Length < 1) { ok = false; break; }
        if (!list0[0].StartsWith("B:")) { ok = false; break; }
```
- 驗證資料存在且型態為 Binary（`B:`），否則視為解析失敗

```csharp
        MHEAD = new int[list0.Length-1];
        for(int i=0; i < MHEAD.Length; i++) {
            MHEAD[i] = sp.BinToInt(list0[1+i]);
            }
```
- `list0.Length - 1` — 排除第一個 token（型態），其餘為 byte 值
- `sp.BinToInt(...)` — 將 Binary hex token 轉為 int，存入 `MHEAD[]` 陣列
- 可解析 `MHEAD[2]` / `MHEAD[3]` 取得原訊息的 Stream/Function，判斷哪筆訊息被設備視為非法

```csharp
        } // end while(ok)
    } // end recv_S9F7
```
- S9F7 無需回覆（Host 收到後應記錄錯誤、通知操作者，不需呼叫 `SendReply`）
