# S1F14 - Establish Communications Request Acknowledge

**Direction:** Different Host and Equipment Use

---

| **S1F14** | Establish Communications Request Acknowledge | Different Host and Equipment Use |

Comment: MDLN and SOFTREV may not be valid unless COMMACK value is 0

## Message Format Definition

### Equipment Send Format (Device Reply)

```
{L:2
  [COMMACK]
  {L:2
    [MDLN]
    [SOFTREV]
  }
}
```

**Format 說明：**
- **格式**（Format）定義了消息的數據結構，使用 SECS-II 列表標記法
- `{L:2}` = 列表，包含 2 個元素
- `[COMMACK]` = 通信確認碼（B:1 = 1 字節二進制）
- 嵌套 `{L:2}` = 內層列表，包含設備型號和軟件版本
  - `[MDLN]` = 設備型號（A:20 = 20 字符 ASCII）
  - `[SOFTREV]` = 軟件版本（A:20 = 20 字符 ASCII）

### Host Send Format (Host Reply)

```
{L:2
  [COMMACK]
  L:0
}
```

**格式說明：**
- `{L:2}` = 列表，包含 2 個元素
- `[COMMACK]` = 通信確認碼
- `L:0` = 空列表（不包含任何元素）

### Equipment Reply Parsing (C# Codebase Example)

```csharp
// S1F14 C# Parse Equipment reply
SecsHost sp=(SecsHost)your_SecsHost;
RcResult rcr = sp.SendSecsMsg(1, 13, true, sendData, true);
string TSN_data;
bool ok=true;
while (ok) {   // break out of loop on error
    if (rcr.rc != 0) {ok=false; break;}
    TSN_data = rcr.result;
    // expect L:2 COMMACK {L:2 MDLN SOFTREV}
    // variables for data items and parsing
    int COMMACK;  	// B:1 (always)  establish communications acknowledgement code
    string MDLN;  	// A:20 (always)  equipment model type
    string SOFTREV;  	// A:20 (always)  software revision
    string [] list0;
    list0 = SecsPort.ListSplit(TSN_data);
    if (list0.Length != 3 || list0[0] != "L:2") { ok=false; break; }
    string [] list1;
    list1 = SecsPort.ListSplit(list0[1]);
    if (list1.Length < 1) { ok = false; break; }
    if (list1[0] != "B:1") { ok = false; break; }
    COMMACK = sp.BinToInt(list1[1]);
    list1 = SecsPort.ListSplit(list0[2]);
    if (list1.Length != 3 || list1[0] != "L:2") { ok=false; break; }
    string [] list2;
    list2 = SecsPort.ListSplit(list1[1]);
    if (list2.Length < 1) { ok = false; break; }
    if (!list2[0].StartsWith("A:")) { ok = false; break; }
    if (list2.Length == 2) { MDLN = list2[1];} else { MDLN = ""; }
    list2 = SecsPort.ListSplit(list1[2]);
    if (list2.Length < 1) { ok = false; break; }
    if (!list2[0].StartsWith("A:")) { ok = false; break; }
    if (list2.Length == 2) { SOFTREV = list2[1];} else { SOFTREV = ""; }
    return;  // parsed ok
    } // end while(ok)
```

**Codebase 實現說明：**
- **SecsPort.ListSplit()**：將 SECS-II 列表格式字符串分割為數組
  - 根據格式定義，設備回覆被分割為 3 個部分（`list0[0]` = "L:2", `list0[1]` = COMMACK, `list0[2]` = 嵌套列表）
- **驗證結構**：檢查類型標記是否符合預期（B:1、L:2、A:20 等）
- **數據提取**：使用 `BinToInt()` 將二進制轉換為整數，字符串直接提取
- **錯誤處理**：任何驗證失敗都設置 `ok=false`，跳出迴圈

### Host Reply Parsing (C# Codebase Example)

```csharp
// S1F14 C# Parse Host reply
SecsPort sp=(SecsPort)your_SecsPort
RcResult rcr = sp.SendSecsMsg(1, 13, true, sendData, true);
string TSN_data;
bool ok=true;
while (ok) {   // break out of loop on error
    if (rcr.rc != 0) {ok=false; break;}
    TSN_data = rcr.result;
    // expect L:2 COMMACK L:0
    // variables for data items and parsing
    int COMMACK;  	// B:1 (always)  establish communications acknowledgement code
    string [] list0;
    list0 = SecsPort.ListSplit(TSN_data);
    if (list0.Length != 3 || list0[0] != "L:2") { ok=false; break; }
    string [] list1;
    list1 = SecsPort.ListSplit(list0[1]);
    if (list1.Length < 1) { ok = false; break; }
    if (list1[0] != "B:1") { ok = false; break; }
    COMMACK = sp.BinToInt(list1[1]);
    list1 = SecsPort.ListSplit(list0[2]);
    if (list1.Length != 1 || list1[0] != "L:0") { ok = false; break; }
    // single token data such as L:0 or U4:0 has been received as expected
    return;  // parsed ok
    } // end while(ok)
```

**Codebase 實現說明：**
- **主機回覆**：比設備回覆簡化，只包含 `COMMACK` 和空列表 `L:0`
- **驗證流程**：同樣檢查結構和類型，但嵌套列表為空（`list1.Length != 1 || list1[0] != "L:0"`）
- **對稱設計**：兩個解析函數遵循相同的驗證模式，展示了 SECS-II 實現的一致性
