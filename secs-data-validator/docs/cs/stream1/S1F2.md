# S1F2 - On Line Data

**Direction:** Different Host and Equipment Use

---

| **S1F2** | On Line Data | Different Host and Equipment Use |

**Comment:** Equipment sends the reply to indicate an online control state, otherwise an abort reply is seen.

## Overview

S1F2 is the reply to S1F1 (On Line Request). This message demonstrates a key SECS-II pattern:
- **Equipment perspective:** When queried if online, respond with identification info (model + version)
- **Host perspective:** Acknowledge receipt with a simple confirmation (empty list)

---

## Message Format Details

### TSN (Token String Notation) Notation Rules

SECS-II message data is encoded as a **token string** (`TSN_data`). Each element uses the format:

```
TypeCode:Count [value1 value2 ...]
```

| Type Code | Meaning              | Example Token     | Parsed Value        |
|-----------|----------------------|-------------------|---------------------|
| `L:N`     | List with N elements | `L:2`             | Container (no value)|
| `A:N`     | ASCII string, N chars| `A:20 MYEQUIP001 `| `"MYEQUIP001"`      |
| `U4:1`    | Unsigned 32-bit int  | `U4:1 12345`      | `12345`             |
| `B:N`     | Binary bytes, N bytes| `B:2 0x01 0x02`   | `[0x01, 0x02]`      |

> **Key rule:** `L:N` is a container — it has no value itself, just N child elements that follow. All other types with `:0` are empty (zero-length).

### `SecsPort.ListSplit()` Behaviour

`SecsPort.ListSplit(token)` splits **one level** of a TSN token:

- **For a list token** like `"L:2 {A:20 ABC} {A:20 1.0}"`:
  - Returns `["L:2", "{A:20 ABC}", "{A:20 1.0}"]`
  - `[0]` = type token (`"L:2"`), `[1..N]` = child tokens (one per list element)
- **For a scalar token** like `"A:20 MYEQUIP001 "`:
  - Returns `["A:20", "MYEQUIP001 "]`
  - `[0]` = type token, `[1]` = the value string

## Equipment Send Format

When equipment replies to host's S1F1 ("Are you online?"):

```
L:2
  MDLN    A:20   Equipment model type (fixed-length ASCII, max 20 chars)
  SOFTREV A:20   Software revision    (fixed-length ASCII, max 20 chars)
```

### Example Wire Data (TSN_data)

```
L:2 {A:20 MODEL-XYZ-1000  } {A:20 V2.3.1              }
```

After `SecsPort.ListSplit()` at each level:

```
list0 = ListSplit(TSN_data)
// list0[0] = "L:2"
// list0[1] = "A:20 MODEL-XYZ-1000  "
// list0[2] = "A:20 V2.3.1              "

list1 = ListSplit(list0[1])
// list1[0] = "A:20"
// list1[1] = "MODEL-XYZ-1000  "   ← MDLN value (trim as needed)

list1 = ListSplit(list0[2])
// list1[0] = "A:20"
// list1[1] = "V2.3.1              "  ← SOFTREV value (trim as needed)
```

### Validation Rules (Equipment reply)

| Check | Condition | Action on Fail |
|-------|-----------|----------------|
| Outer list count | `list0.Length == 3` | `ok = false; break` |
| Outer list type | `list0[0] == "L:2"` | `ok = false; break` |
| MDLN type prefix | `list1[0].StartsWith("A:")` | `ok = false; break` |
| MDLN value present | `list1.Length == 2` | Use `""` as fallback |
| SOFTREV type prefix | `list1[0].StartsWith("A:")` | `ok = false; break` |
| SOFTREV value present | `list1.Length == 2` | Use `""` as fallback |

## Host Send Format

When host replies to equipment's S1F2 acknowledgment ("Got it"):

```
L:0    (empty list — no data needed)
```

### Example Wire Data (TSN_data)

```
L:0
```

After `SecsPort.ListSplit()`:

```
list0 = ListSplit(TSN_data)
// list0[0] = "L:0"   ← only one token, no children
// list0.Length == 1
```

### Validation Rules (Host reply)

| Check | Condition | Action on Fail |
|-------|-----------|----------------|
| Single token only | `list0.Length == 1` | `ok = false; break` |
| Empty list type | `list0[0] == "L:0"` | `ok = false; break` |

## C# Parse Code

### Parse Equipment Reply (Host receives from Equipment)

```csharp
// S1F2 C# Parse Equipment reply
SecsHost sp = (SecsHost)your_SecsHost;
RcResult rcr = sp.SendSecsMsg(1, 1, true, sendData, true);
string TSN_data;
bool ok = true;
while (ok) {   // break out of loop on error
    if (rcr.rc != 0) { ok = false; break; }
    TSN_data = rcr.result;

    // Expect: L:2 {A:20 <MDLN>} {A:20 <SOFTREV>}
    string MDLN;     // A:20 (always)  equipment model type
    string SOFTREV;  // A:20 (always)  software revision

    string[] list0 = SecsPort.ListSplit(TSN_data);
    if (list0.Length != 3 || list0[0] != "L:2") { ok = false; break; }

    // Parse MDLN (list0[1])
    string[] list1 = SecsPort.ListSplit(list0[1]);
    if (list1.Length < 1) { ok = false; break; }
    if (!list1[0].StartsWith("A:")) { ok = false; break; }
    MDLN = (list1.Length == 2) ? list1[1] : "";

    // Parse SOFTREV (list0[2])
    list1 = SecsPort.ListSplit(list0[2]);
    if (list1.Length < 1) { ok = false; break; }
    if (!list1[0].StartsWith("A:")) { ok = false; break; }
    SOFTREV = (list1.Length == 2) ? list1[1] : "";

    // Use MDLN and SOFTREV here
    // e.g. Console.WriteLine($"Equipment: {MDLN.Trim()}, SW: {SOFTREV.Trim()}");
    return;  // parsed ok
} // end while(ok)
// parse error handling here
```

### Parse Host Reply (Equipment receives from Host)

```csharp
// S1F2 C# Parse Host reply
SecsPort sp = (SecsPort)your_SecsPort;
RcResult rcr = sp.SendSecsMsg(1, 1, true, sendData, true);
string TSN_data;
bool ok = true;
while (ok) {   // break out of loop on error
    if (rcr.rc != 0) { ok = false; break; }
    TSN_data = rcr.result;

    // Expect: L:0 (empty list — host acknowledged, no data)
    string[] list0 = SecsPort.ListSplit(TSN_data);
    if (list0.Length != 1 || list0[0] != "L:0") { ok = false; break; }

    // Empty list confirmed; host is online
    return;  // parsed ok
} // end while(ok)
// parse error handling here
```
