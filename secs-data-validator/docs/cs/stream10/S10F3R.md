# S10F3[R] - Terminal Display, Single

**Direction:** Sent by Host Only  
**Reply:** S10F4 (Terminal Display Single Acknowledge)  
**Stream 10:** Terminal Services — 用於 Host 與 Equipment 上掛載的操作員終端機（Operator Terminal）之間進行文字通訊。

---

## 用意與目的

`S10F3` 是 SECS-II **Stream 10（Terminal Services）** 的核心訊息之一，由 **Host 傳送給 Equipment**，要求 Equipment 將一段文字顯示在指定的終端機（Terminal）螢幕上。

### 使用情境

| 情境 | 說明 |
|------|------|
| 操作員通知 | Host 需要在設備前台顯示警示訊息或操作指示 |
| 批次資訊傳遞 | 顯示當前批次（Lot）名稱、配方（Recipe）或工單（Job Order）資訊 |
| 多終端機管理 | 一台 Equipment 可以掛載多個終端機，透過 `TID` 指定目標 |
| 遠端操作提示 | Host 系統（MES/CIM）在遠端向操作員發出提示 |

### 訊息流程

```
Host                        Equipment
  |                              |
  |  S10F3 (TID=0, TEXT="...")  |
  | ─────────────────────────►  |
  |                              |  顯示文字於 Terminal #0
  |  S10F4 (ACKC10)             |
  | ◄─────────────────────────  |
```

- Host 送出 `S10F3`，Equipment **必須**回覆 `S10F4`（包含 `ACKC10` 確認碼）
- 若資料格式錯誤，Equipment 應送出 `S9F7`（Illegal Data）而非 `S10F4`
- `[R]` 標記表示此訊息需要回覆（Reply bit = 1）

## Format 說明

```
S10F3
{L:2
  TID
  TEXT }
```

| 欄位 | 型別 | 長度 | 說明 |
|------|------|------|------|
| `L:2` | List | 固定 2 個元素 | 外層 List 結構 |
| [`TID`](http://hume.com/secs/items.html#TID) | `B:1` | 固定 1 byte | Terminal ID，終端機編號（0–255） |
| [`TEXT`](http://hume.com/secs/items.html#TEXT) | `A:n` | 可變長度 | 要顯示的文字，ASCII 字串；`A:0` 表示空字串 |

- 大括號 `{ }` 表示一個 SECS-II List 資料結構
- `TID = 0` 通常代表主要操作員終端機（Primary Terminal）
- `TEXT` 無標準最大長度限制（dmhclient 文件中標記為 `A:120` 係指常見實作慣例）

---

## 資料格式與型態說明

### TID — Terminal ID (`B:1`)

`B` 型別（Binary）以 1 個 byte 表示終端機編號，範圍 0–255。

| TID 值 | 慣例意義 |
|--------|----------|
| 0 | 主終端機（Primary / Default Terminal） |
| 1–255 | 第 2 至第 256 台附加終端機 |

**TSN 格式範例：**

```
B:1 0x00    → TID = 0（主終端機）
B:1 0x01    → TID = 1
B:1 0xFF    → TID = 255
```

### TEXT — 顯示文字 (`A:n`)

`A` 型別（ASCII）為可變長度字串。

| 狀況 | TSN 格式 | C# 解析結果 |
|------|----------|-------------|
| 一般文字 | `A:13 Hello, World!` | `TEXT = "Hello, World!"` |
| 空字串 | `A:0` 或 `A` | `TEXT = ""` |
| 中文/Unicode | **不支援**（純 ASCII） | 需轉換為 ASCII 表示 |

## 完整 TSN 訊息範例

### 範例 1：向終端機 0 顯示文字

```
L:2
  B:1 0x00
  A:22 Lot ABC-001 started OK
```

- `TID = 0`（主終端機）
- `TEXT = "Lot ABC-001 started OK"`

### 範例 2：向終端機 1 顯示警示

```
L:2
  B:1 0x01
  A:16 ALARM: Door Open!
```

- `TID = 1`（第二台終端機）
- `TEXT = "ALARM: Door Open!"`

### 範例 3：清空終端機顯示（空字串）

```
L:2
  B:1 0x00
  A:0
```

- `TID = 0`
- `TEXT = ""`（設備應清除畫面或不顯示任何文字）

## S10F4 Reply — ACKC10 確認碼

Equipment 收到 `S10F3` 後必須回覆 `S10F4`，內含 `ACKC10`：

| ACKC10 值 | 意義 |
|-----------|------|
| `0x00` | Acknowledged（已確認，文字已顯示或排程顯示） |
| `0x01` | Terminal Not Available（終端機不可用） |
| 其他 | 依設備定義 |

## Codebase 說明（C# / dmhclient）

| 方法 / 變數 | 說明 |
|-------------|------|
| `sp.MessageTypeAdd(10, 3, ...)` | 向 SecsPort 註冊 S10F3 的接收 handler |
| `SecsPort.ListSplit(TSN_data)` | 將 TSN 字串解析為字串陣列；`[0]` 為型別標籤，後續為資料值 |
| `list0[0] != "L:2"` | 驗證第一層結構為 2 元素 List |
| `list1[0] != "B:1"` | 驗證 TID 欄位型別為 binary 1 byte |
| `sp.BinToInt(list1[1])` | 將 binary 字串轉換為整數（取得 TID 數值） |
| `list1.Length == 2` | 判斷 TEXT 是否有內容（長度 2 = 型別標籤 + 值；長度 1 = 空字串） |
| `sp.SendReply(10, 4, transID, reply)` | 送出 S10F4 reply（Terminal Display Single Acknowledge） |
| `sp.SendS9(7, header)` | 資料格式錯誤時送出 S9F7（Illegal Data） |

## C# 實作範例（dmhclient 風格）

### 接收與解析 S10F3

```csharp
// S10F3 C# Receive message - add next line to setup
// SecsPort sp = your_SecsPort;
sp.MessageTypeAdd(10, 3, new SecsMsgReceiveDelegate(recv_S10F3));

void recv_S10F3(object sender, int stream, int function, bool send_reply,
                int transID, string TSN_data, string header)
{
    SecsPort sp = (SecsPort)sender;
    bool ok = true;

    while (ok) {   // break out of loop on error
        // expect L:2 TID TEXT
        int TID;      // B:1 (always)  terminal ID, 0–255
        string TEXT;  // A:n (varies)  line of text to display

        string[] list0 = SecsPort.ListSplit(TSN_data);
        // list0[0] = "L:2",  list0[1] = TID element,  list0[2] = TEXT element
        if (list0.Length != 3 || list0[0] != "L:2") { ok = false; break; }

        string[] list1 = SecsPort.ListSplit(list0[1]);
        // list1[0] = "B:1",  list1[1] = binary byte value
        if (list1.Length < 1) { ok = false; break; }
        if (list1[0] != "B:1") { ok = false; break; }
        TID = sp.BinToInt(list1[1]);   // e.g. "0x00" → 0

        list1 = SecsPort.ListSplit(list0[2]);
        // list1[0] = "A:n"
        // list1.Length == 2  →  "A:22" + "Lot ABC-001 started OK"
        // list1.Length == 1  →  "A:0"  (empty string)
        if (list1.Length < 1) { ok = false; break; }
        TEXT = (list1.Length == 2) ? list1[1] : "";

        // --- application logic ---
        Console.WriteLine($"[Terminal {TID}] {TEXT}");
        // UpdateTerminalDisplay(TID, TEXT);

        if (send_reply) {
            string ackc10 = sp.IntToBin(0, 1);  // 0x00 = Acknowledged
            string reply = $"B:1 {ackc10}";
            sp.SendReply(10, 4, transID, reply);
        }
        return;
    } // end while(ok)

    // bad data → S9F7 Illegal Data
    sp.SendS9(7, header);
}
```

### 重點解析步驟說明

```
TSN_data 範例：
"L:2 {B:1 0x00} {A:22 Lot ABC-001 started OK}"

ListSplit(TSN_data):
  list0[0] = "L:2"
  list0[1] = "B:1 0x00"        ← TID 元素
  list0[2] = "A:22 Lot ABC..."  ← TEXT 元素

ListSplit(list0[1]):
  list1[0] = "B:1"
  list1[1] = "0x00"            → BinToInt("0x00") = 0

ListSplit(list0[2]):
  list1[0] = "A:22"
  list1[1] = "Lot ABC-001 started OK"  → TEXT = this value
```

## secs4net 風格實作（現代 C#）

若使用本 codebase 的 secs4net v2.4.4，實作為 `IMessageHandler`：

```csharp
using Secs4Net;

public class S10F3Handler : IMessageHandler
{
    public int Stream => 10;
    public int Function => 3;

    public async ValueTask HandleAsync(PrimaryMessageWrapper wrapper)
    {
        var msg = wrapper.PrimaryMessage;

        // msg.SecsItem: L:2 { B TID, A TEXT }
        var body = msg.SecsItem;                     // L:2
        var tidItem  = body[0];                      // B:1
        var textItem = body[1];                      // A:n

        byte tid  = tidItem.FirstValue<byte>();      // 0–255
        string text = textItem.Count > 0             // A:0 = empty
                    ? textItem.GetString()
                    : string.Empty;

        Console.WriteLine($"[Terminal {tid}] {text}");

        // Reply S10F4 with ACKC10 = 0 (Acknowledged)
        await wrapper.TryReplyAsync(
            new SecsMessage(10, 4,
                Item.B(0x00)));   // ACKC10
    }
}
```

**secs4net Item API 快速參考：**

| 操作 | 語法 |
|------|------|
| 建立 Binary item | `Item.B(0x00)` |
| 建立 ASCII item | `Item.A("Hello")` |
| 建立空 ASCII | `Item.A(string.Empty)` |
| 建立 List | `Item.L(tidItem, textItem)` |
| 讀取 byte 值 | `item.FirstValue<byte>()` |
| 讀取字串 | `item.GetString()` |
| 讀取 item 長度 | `item.Count` |

## 相關訊息

| 訊息 | 方向 | 說明 |
|------|------|------|
| `S10F1` | Equipment → Host | 終端機文字傳送（Equipment 發給 Host） |
| `S10F2` | Host → Equipment | S10F1 的 ACK |
| **`S10F3`** | Host → Equipment | **終端機顯示（本訊息）** |
| `S10F4` | Equipment → Host | S10F3 的 ACK（ACKC10） |
| `S10F5` | Host → Equipment | 多行終端機顯示（Multi-Block） |
| `S10F6` | Equipment → Host | S10F5 的 ACK |
| `S9F7`  | Equipment → Host | Illegal Data（格式錯誤時回覆） |
